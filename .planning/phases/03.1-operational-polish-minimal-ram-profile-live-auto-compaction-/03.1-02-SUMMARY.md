# Plan 03.1-02 SUMMARY — SearxNG + web_search + web_fetch bypass + 3-state badge

## One-line

Shipped self-hosted SearxNG as an auditable loopback-only outbound harness; added `web_search(query, max_results)` tool that rotates Google/DDG/Brave/Bing/Startpage with automatic rate-limit fallback; swapped web_fetch from strict allowlist-only to returned-URL bypass (exact-URL match; SSRF guard against same-host path mutation); extended offline badge to 3 states; split air-gap CI into strict-inference + permissive-research-egress.

## What shipped

1. **SearxNG stack** (`observability/searxng/`) — Docker Compose sibling of Langfuse; SearxNG + its own dedicated Redis (self-contained per plan-checker WARN 1 to avoid cross-stack coupling); digest-pinned images; loopback-bound `127.0.0.1:8888`; 90s health gate; openssl-generated secrets on first boot. Stack up in ~60s, stable.

2. **`web_search` tool** (`packages/emmy-tools/src/web-search.ts`) — SearxNG JSON API wrapper at `/search?format=json`. Returns `{title, url, snippet, engine}[]`. 3 kill-switches (profile `enabled=false`, `EMMY_WEB_SEARCH=off`, `EMMY_TELEMETRY=off`). 10-call/turn rate limit (T-03.1-02-03) via counter reset on pi `turn_end` (plan-checker WARN 2 addressed). ToolError-shaped fallback on any failure (never throws).

3. **web_fetch returned-URL bypass** (`packages/emmy-tools/src/web-fetch-allowlist.ts`) — `RecentSearchUrlStore` with 5-min TTL. `recordSearchUrl` fires from web_search's success path. `enforceWebFetchAllowlist` now checks `recentSearchUrls?.has(exact_url)` BEFORE the allowlist (bypass wins). EXACT URL match only — same host, different path is DENIED (T-03.1-02-02 SSRF guard).

4. **3-state offline badge** (`packages/emmy-ux/src/offline-badge.ts`) — green `OFFLINE OK` when SearxNG down/disabled; yellow `LOCAL LLM · WEB` (ANSI 33) on first web_search success; red `CLOUD INFERENCE` reserved for the never-expected case of inference going non-loopback. `OfflineAuditResult` gained `badge_state` field; session.ts wires `webSearchOnSuccess → flipToYellow`, `webSearchOnFallback → flipToGreen`.

5. **Profile v3.1 harness.yaml** adds `tools.web_search: { enabled: true, base_url, max_results_default, rate_limit_per_turn, timeout_ms }` + `tools.web_fetch.search_bypass_ttl_ms: 300000`. Pydantic schema extended with `WebSearchConfig` + `WebFetchConfig.search_bypass_ttl_ms`. v1/v2/v3 validate unchanged.

6. **Air-gap CI split** — `ci_verify_phase3.py` stays strict (local-inference gate, unchanged). New `ci_verify_research_egress.py` permits SearxNG outbound but hard-denies 12 cloud-inference endpoints (`api.openai.com`, `api.anthropic.com`, `generativelanguage.googleapis.com`, AWS Bedrock, etc.). Both pass `--dry-run` exit 0.

7. **pi.on("turn_end") rate-limit reset** — registered at session-start in pi-emmy-extension.ts, unconditional (not gated on telemetry) so the `_turnSearchCount` counter doesn't accumulate across long sessions.

## Walkthrough-discovered integration fixes

Three gaps surfaced when the agent tried to actually USE web_search live. Plan-checker didn't catch them because unit tests mocked the prompt-assembly and profile-loader layers. Same lesson as Plans 03-05 and 03-08: **live walkthrough is load-bearing for integration class.**

1. **`profiles/qwen3.6-35b-a3b/v3.1/prompts/tool_descriptions.md`** — cloned from v3 without adding a web_search section. Fixed.
2. **`packages/emmy-ux/src/session.ts` prompt assembly** — `toolDefsText` was a hardcoded 8-tool list in the session.ts prompt-assembly code (NOT read from the profile's tool_descriptions.md file, which is a separate artifact). Added conditional `web_search` line gated on profile + kill-switches.
3. **`packages/emmy-ux/src/profile-loader.ts`** — never parsed `tools.web_search` from harness.yaml, so `opts.profile.harness.tools.web_search` was `undefined` downstream. Added `WebSearchBlock` parser + emit.

After fixes, agent successfully invoked `web_search` on live emmy-serve + live SearxNG and correctly reported: `"The latest Bun release version is 1.3.13, as indicated on both the Bun website and npm."` (ground truth — Bun 1.3.13 published 1 day prior per SearxNG results from google + npm registry).
