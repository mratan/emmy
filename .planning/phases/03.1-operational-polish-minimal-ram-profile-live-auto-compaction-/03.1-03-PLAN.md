    ## Log locations
    - emmy-serve: `/var/log/emmy-serve/` (inside container) — stdout/stderr via `docker logs`
    - JSONL event stream: `runs/<session_id>/events.jsonl` (authoritative)

    - Langfuse UI: http://localhost:3000 (after `start_observability.sh`)

    ## Common errors and resolutions

    ### "Profile hash mismatch"
    - Symptom: `uv run emmy profile validate` fails with hash mismatch.
    - Cause: profile YAML was edited in place (D-02 violation) or hash not recomputed after a sanctioned sibling creation.
    - Fix: `uv run emmy profile hash --write profiles/qwen3.6-35b-a3b/<version>/` to recompute.
    - Never edit v1, v2, or v3 — they are certified at prior phase closeouts; v3.1 is the current mutable-during-a-plan sibling.

    ### "SessionTooFullError"
    - Symptom: D-12 fail-loud after the 114688-token hard wall.
    - Cause: a single turn's assembled prompt + tool results exceeds the ceiling; compaction ran but didn't free enough.
    - Fix: `/clear` to start fresh session. Consider lowering per-turn file-read scope in your next session.

    ### "SearxNG engine X is not returning results"
    - Cause: upstream rate-limit. SearxNG auto-rotates to the next engine; if all four rate-limit simultaneously, web_search returns ToolError.
    - Fix: wait 60-120s; or disable the rate-limited engine in `observability/searxng/settings.yml` (re-run `start_searxng.sh` to pick up).

    ### "EMMY_WEB_SEARCH=off but badge still shows yellow"
    - Cause: env was set AFTER pi-emmy started.
    - Fix: restart pi-emmy after setting the env variable.

    ### "free -h shows <40G available"
    - Cause: gpu_memory_utilization is higher than recommended for your workload.
    - Fix: create a v3.2 profile sibling with lower value; iterate. See Plan 03.1-01 for precedent.

    ### "Langfuse UI won't load (502)"
    - Cause: langfuse-web healthcheck times out on first boot while Prisma migrations run.
    - Fix: wait 90s after `start_observability.sh`; the healthcheck start_period is set accordingly. If still failing, `docker compose -f observability/langfuse/docker-compose.yaml logs langfuse-web | tail -50`.

    ## Langfuse API-key rotation
