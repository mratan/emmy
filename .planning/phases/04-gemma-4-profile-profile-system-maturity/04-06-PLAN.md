scripts/find_kv_budget.py:
  --profile PATH        (path to the bundle, e.g. profiles/gemma-4-26b-a4b-it/v1/)
  --start FLOAT         (starting gpu_memory_utilization, default 0.75)
  --step FLOAT          (bisection step, default 0.02)
  --max FLOAT           (upper bound, default 0.95)
  --port INT            (vLLM port, default 8002)
  Writes measured gpu_memory_utilization to the profile's serving.yaml on success; bails + restores on any preemption.

scripts/thermal_replay.py:
  --profile PATH        (path to bundle)
  --record-floors       (first run; writes floors to PROFILE_NOTES.md frontmatter)
  --assert-floors       (subsequent runs; asserts measured >= recorded floors; exit non-zero if not)
  --duration 7200       (default 2 hours)

pi-emmy (Plan 04-03 produced):
  Start session: `pi-emmy`
  Inside session: `/profile <name>[@<variant>]` → triggers child process swap
  `/quit` to exit

emmy-serve swap-profile (Plan 04-02 produced — direct CLI):
  `uv run emmy-serve swap-profile --from profiles/<old> --to profiles/<new> --port 8002`
  Useful for scripted testing; pi-emmy /profile is the user-facing path.
