## V6 byte-identical evidence

The snapshot-fidelity test walks `liveGlobalAbs` after revert and asserts
size + sha256 of every entry equals the pre-apply state.  Concretely
(global-only scenario):

- Pre-apply: 2 operator files (`operator-1.md`, `operator-2.md`) at
  random shas
- Apply: live root mirrors T0_GLOBAL (4 files at known content)
- Run dispatcher: view, create, str_replace, insert, rename, delete
  exercise the live root
- Revert: live root walks back to 2 entries with byte-identical content
  to pre-apply

Same invariant holds in the transactional project+global case.

## Precedence rule — 3-tier verification

`resolveMemoryConfig` test cases prove the env > profile > flag-default
ordering:

1. **env wins:** `EMMY_MEMORY_OVERRIDE_PROJECT=/tmp/x` + profile says
   `profile_proj` → resolved `project_root === "/tmp/x"`.
2. **profile wins (no env):** profile says `profile_proj`, no env →
   resolved `project_root === "profile_proj"`.
3. **DEFAULT_MEMORY_CONFIG fallback:** no profile + no env → resolved
   matches `DEFAULT_MEMORY_CONFIG`.
4. **Empty-string env is treated as unset** (matches CLAUDE.md rule for
   URL precedence).

`--no-memory` short-circuit takes priority over both: returns
`{enabled: false}` regardless of env or profile.
