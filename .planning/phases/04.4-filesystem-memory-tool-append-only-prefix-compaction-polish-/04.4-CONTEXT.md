## Phase Boundary

This phase ships two co-phased pieces of work that share an `execute-phase` wave but **share no code**:

**Half A — Filesystem memory tool.** Adopt Anthropic's `memory_20250818` tool verbatim. Six commands (view / create / str_replace / insert / delete / rename) over a virtual `/memories` root resolved per-scope to `<repo>/.emmy/notes/` (project) or `~/.emmy/memory/` (global). Path-traversal guard. Profile-owned config (`harness.yaml memory.*`). Reproducibility hooks for eval (`--no-memory`, `--memory-snapshot`, `EMMY_MEMORY_OVERRIDE_*`). Telemetry per op. Air-gap clean.

**Half B — Compaction polish.** Codify the append-only-prefix invariant (D-3X — system-prompt prefix never mutates mid-session) as a project-level decision and audit it via per-request `emmy.prefix.hash` OTel attribute. Replace pi's default compaction prompt with Emmy's terse v1 (~35 tokens) measurement-driven prompt across all 4 shipped profiles. Confirm `/compact [reason]` and `/clear` slash-command stubs from Phase 3.1 are wired and verified.

**What this phase does NOT include** (deliberately, per pre-phase decisions):
- Vector embeddings or RAG-over-notes (filesystem-only — Letta LoCoMo evidence)
- Automatic note generation (model decides when to write)
- Cross-machine sync (single-user-local thesis)
- Encryption at rest (filesystem provides)
- Multiple memory backends (YAGNI)
- An MCP server wrapper (native pi tool first)
- Hierarchical / multi-level summarization
- Compaction-time auto-write to memory
- Forced compaction-prompt sectioning (goals/decisions/blockers as named fields — pi minimalism: model knows what a summary is)
- Soft-threshold tuning per profile (Phase 5 eval decision)

## Implementation Decisions (LOCKED)

### Memory tool — adopted surface (Anthropic `memory_20250818`)
- **Six commands**, schemas lifted directly from Anthropic's spec:
  - `view` — directory tree listing OR numbered file lines (1-indexed); optional `view_range: [start, end]`
  - `create` — errors if file already exists (deliberate; forces `str_replace` for updates)
  - `str_replace` — `old_str` must match exactly once; on collision, return line numbers of all matches so model can resubmit with more context
  - `insert` — at `insert_line: N`
  - `delete` — non-recursive on directories (must be empty); returns contained-files list on non-empty
  - `rename` — files and dirs
- **Path semantics:** all paths logical under virtual root `/memories`. Path traversal blocked: reject `..`, `..\\`, `%2e%2e%2f`, or any path that resolves outside configured root after symlink expansion. Return typed error.

### Memory tool — Emmy-specific extensions
- **Two scopes via path prefix** (NOT separate tools):
  - `/memories/project/...` → `<repo>/.emmy/notes/` (per-project, gitignorable)
  - `/memories/global/...` → `~/.emmy/memory/` (per-user, cross-project)
  - `view /memories` returns `project/` and `global/` as top-level entries
  - Split is convention enforced by path validation
- **Profile schema** (added to `harness.yaml` for all 4 shipped profiles):
  ```yaml
  memory:
    enabled: true
    project_root: ".emmy/notes"            # relative to cwd; null = disable scope
    global_root: "~/.emmy/memory"          # ~-prefixed; null = disable scope
    read_at_session_start: true
    max_file_bytes: 65536                  # per-file write cap
    max_total_bytes: 10_485_760            # per-scope cap (10 MiB)
    blocked_extensions: [".env", ".key", ".pem"]
  ```
- **Reproducibility hooks:**
  - `--no-memory` flag → disables tool entirely; tool description omitted; instinct prompt skipped
  - `--memory-snapshot DIR` → mirror DIR into live roots before run; restore originals after; atomic per scope
  - `EMMY_MEMORY_OVERRIDE_PROJECT` / `EMMY_MEMORY_OVERRIDE_GLOBAL` env vars → override scope roots; precedence is **env > profile > flag** (matches CLAUDE.md URL-config rule)
  - **Provenance stamp:** every memory op's OTel event includes `{profile.id, profile.version, profile.hash, memory.scope, memory.path}`
- **Air-gap compatibility:** local-fs-only. No network, no IPC outside harness process. Air-gap CI gains a 100-op stress run check: zero network syscalls.

### Memory tool — model-facing description (v1, ~45 tokens; minimum-viable per pi minimalism)
> `memory` — read and write notes that persist across sessions. Two scopes: `/memories/project/...` for repo-specific knowledge, `/memories/global/...` for cross-project preferences. Check what's there before non-trivial work; write notes only when a discovery would help a future session.

**Calibration protocol:** measurement-driven expansion only. After v1 description ships, run V1–V3 verification (see Success Criteria below). If adoption < 60% OR rot protection < 100%, expand description with sterner language. Do NOT pre-emptively bloat.

### Memory tool — telemetry (OTel GenAI semconv)
Every op emits one span under the active tool span:
```
gen_ai.tool.name = "memory"
gen_ai.tool.call.arguments = "<redacted file_text if blocked_extensions match>"
emmy.memory.command = "view" | "create" | ...
emmy.memory.scope = "project" | "global"
emmy.memory.path = "/memories/project/notes/conventions.md"
emmy.memory.bytes = <int>      # for read/write
emmy.memory.result = "ok" | "exists" | "not_found" | "quota_exceeded" | "traversal_blocked" | "ambiguous_match"
```
Plus headless JSON envelope counters: `telemetry.memory_ops.{view,create,str_replace,insert,delete,rename}`, `telemetry.memory_bytes_{read,written}`.

### Compaction — append-only-prefix invariant (D-3X, NEW project-level decision)
**The system-prompt prefix sent to vLLM never mutates within a session.** Compaction replaces conversation-body turns; never edits system prompt, tool descriptions, project preamble (CLAUDE.md / AGENTS.md), or any pre-turn content.

**Why (load-bearing):**
1. vLLM automatic prefix caching is the largest free-money win; any prefix mutation invalidates blocks → full recompute on next request → KV pressure → preemption risk.
2. Cognition's 2026 "clean context for verifiers" lesson.
3. Reproducibility — prefix mutations interleaved with compactions create non-deterministic histories.

**What this allows:**
- Pi's `before_provider_request` per-request payload mutations (Emmy's thinking-disable, reactive grammar) — these mutate per-request features, not the conversation prefix
- Pi's existing compaction body replacement
- Custom messages appended after system prompt

**What this forbids:**
- Re-rendering system prompt mid-session
- Re-ordering tool descriptions mid-session
- Mutating CLAUDE.md / AGENTS.md and re-loading
- Adding "session summary so far" to system prompt at compaction time

**Enforcement:**
1. One-time audit of `before_provider_request` and other prefix-touching code
2. Per-request OTel attribute `emmy.prefix.hash = sha256(system_prompt_bytes)` — assertion: any change in prefix.hash within one session-id without a profile reload = violation
3. V3 verification: hash equality at turn 0 and turn N (post-compaction)

### Compaction — v1 prompt (`prompts/compact.md`)
Replaces pi's default. ~35 tokens. Per pi minimalism principle (Pitfall #5 "more prompting" trap), ship minimum, measure, expand only if needed.

```
Summarize the conversation above so a fresh context can resume the work. Preserve: explicit goals, decisions made, errors and their resolutions, files modified and their final state. Drop: dead-end exploration, redundant tool calls, transient state.
```

Updated for **all 4 shipped profiles**: qwen3.6-35b-a3b@v3.1, qwen3.6-27b@v1.1, gemma-4-26b-a4b-it@v2, gemma-4-31b-it@v1.1.

**Calibration plan (V2):** 10 multi-turn coding sessions with organic compaction. Hand-rate summaries on three axes:
- Goal preservation (≥ 80%)
- Decision preservation (≥ 70%)
- Error preservation (100% — only critical dimension)

If any target misses, revise `prompts/compact.md` to add the missing axis explicitly. **Do not pre-emptively name all three axes in v1.**

### Compaction — slash commands (Phase 3.1 finalization)
- **`/compact [reason]`** — manual compaction trigger. Calls `AgentSession.compact({customInstructions: composeReasonPrompt(reason)})`. `reason` appended to v1 prompt as single `Reason: <user text>` line. No structural change. Telemetry: `emmy.compaction.trigger` with `trigger_kind: "manual"`.
- **`/clear`** — full reset. Disposes current session, instantiates new with same profile. Memory dir, sub-agent state, OTel parent trace_id do NOT carry over.
- **Both are headless-mode-incompatible** by design. Headless's `--max-turns` + auto-compaction is the equivalent.

### Compaction — what we keep from pi-mono (do not rebuild)
- `shouldCompact()` predicate (pi-coding-agent core/compaction)
- `compact()` round-trip with token-counting and message-rewriting
- `SessionCompactionEntry` JSONL format (already includes `customInstructions`, `tokensBefore`, `tokensAfter`, `summary`)
- Auto-retry on transient summarizer failures
- D-16 structured-pruning fallback (already implemented in `@emmy/context`)

### Profile schema — NO new context.compaction fields
Existing schema is the right surface:
```yaml
context:
  max_input_tokens: 114688
  compaction:
    soft_threshold_pct: 0.75
    preserve_recent_turns: 5
    preserve_tool_results: error_only
    summarization_prompt_path: prompts/compact.md
```
If V1–V2 verification shows v1 prompt misses, **update `prompts/compact.md`, not the schema.**

### Pitfall #22 awareness (architecture-aware verification)
H9 spike (2026-04-26) discovered vLLM prefix caching is silently 0% hit rate on Mamba-hybrid Qwen 3.6 35B-A3B even with `enable_prefix_caching: True` (vLLM marks it "experimental" for Mamba layers). V4 (prefix-cache hit rate after compaction) must be **architecture-aware per profile**:
- Attention-only (Qwen 27B dense, Gemma 31B dense): hard ≥ 80% target
- Mamba-hybrid (Qwen 35B-A3B, Gemma 26B-A4B): "measure and acknowledge" pass criterion (no false-fail), with auto-detection via boot-log warning text

- Internal directory layout *within* `packages/emmy-tools/src/memory/` is a convention — files listed are proposed; planner may consolidate or split based on emergent module size, as long as the test suite layout matches command-by-command.

- Concurrent-write race handling between `view` and `str_replace` — v0 acceptance is best-effort with `ambiguous_match` retry (no flock). Planner may strengthen if cheap, may leave as-is.
