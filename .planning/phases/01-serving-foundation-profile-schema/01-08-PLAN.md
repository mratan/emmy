```python
"""Validate an airgap-report.json artifact produced by .github/workflows/airgap.yml.

Plan 01-08 certification helper. Used by scripts/verify_airgap_ci.sh.

The artifact shape is defined by emmy_serve.airgap.validator._dump_report:
{
  "ts": "<ISO>",
  "container": "<name>",
  "passes": true | false,
  "failures": [str, ...],                 # empty iff passes
  "layers": [                             # 4 entries, one per layer a/b/c/d
    {"layer": "a"|"b"|"c"|"d", "name": str, "passed": bool,
     "detail": str, "commands_run": [str, ...]}
  ]
}
"""
from __future__ import annotations

import argparse
import json
import sys
from pathlib import Path

_REQUIRED_LAYERS = {"a", "b", "c", "d"}


def validate_airgap_report(report: dict) -> tuple[bool, list[str]]:
    """Return (ok, reasons). ok=True iff the report is a green run."""
    reasons: list[str] = []
    if not isinstance(report, dict):
        return False, ["report is not a dict"]

    if report.get("passes") is not True:
        reasons.append(f"passes={report.get('passes')!r} (expected True)")

    failures = report.get("failures", None)
    if failures is None or failures:
        reasons.append(f"failures is non-empty: {failures}")

    layers = report.get("layers") or []
    present_layers = {layer.get("layer") for layer in layers if isinstance(layer, dict)}
    missing = _REQUIRED_LAYERS - present_layers
    if missing:
        reasons.append(f"missing layers: {sorted(missing)}")

    for layer in layers:
        if not isinstance(layer, dict):
            reasons.append(f"non-dict layer entry: {layer!r}")
            continue
        if not layer.get("passed"):
            reasons.append(
                f"layer ({layer.get('layer')}) {layer.get('name')} not passed: {layer.get('detail')}"
            )

    return (not reasons), reasons


def main(argv: list[str] | None = None) -> int:
    ap = argparse.ArgumentParser(prog="ci_verify")
    ap.add_argument(
        "--from-file",
        required=True,
        help="path to airgap-report.json downloaded from the workflow artifact",
    )
    args = ap.parse_args(argv)

    path = Path(args.from_file)
    if not path.exists():
        print(f"error: file not found: {path}", file=sys.stderr)
        return 2
    try:
        report = json.loads(path.read_text(encoding="utf-8"))
    except json.JSONDecodeError as e:
        print(f"error: invalid JSON in {path}: {e}", file=sys.stderr)
        return 2

    ok, reasons = validate_airgap_report(report)
    if ok:
        print(f"airgap-report OK: passes={report.get('passes')}, 4 layers green, failures=[]")
        print(f"  ts: {report.get('ts')}")
        print(f"  container: {report.get('container')}")
        return 0
    print("airgap-report FAILED validation:", file=sys.stderr)
    for r in reasons:
        print(f"  - {r}", file=sys.stderr)
    return 1


if __name__ == "__main__":
    sys.exit(main())
```

#!/usr/bin/env bash
# scripts/verify_airgap_ci.sh — Plan 01-08 SC-4 certification verifier.
#
# Downloads the airgap-report artifact from the most recent workflow run on
# the current branch (via `gh run download`), validates via the Python helper
# emmy_serve.airgap.ci_verify, and exits 0 iff passes=true + 4 layers green.
#
# Usage:
#   ./scripts/verify_airgap_ci.sh                               # auto-download via gh
#   ./scripts/verify_airgap_ci.sh --from-file /path/to/airgap-report.json  # manual fallback

set -euo pipefail

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
cd "$ROOT_DIR"

mode="gh"
from_file=""
if [[ $# -gt 0 ]]; then
  case "$1" in
    --from-file) mode="file"; from_file="$2"; shift 2 ;;
    -h|--help) sed -n '1,20p' "$0"; exit 0 ;;
    *) echo "unknown arg: $1" >&2; exit 2 ;;
  esac
fi

if [[ "$mode" == "gh" ]]; then
  if ! command -v gh >/dev/null 2>&1; then
    echo "ERROR: 'gh' CLI not installed. Install gh OR use --from-file <path>." >&2
    exit 2
  fi
  branch="$(git rev-parse --abbrev-ref HEAD)"
  run_id="$(gh run list --branch "$branch" --workflow airgap.yml --limit 1 --json databaseId -q '.[0].databaseId' 2>/dev/null || true)"
  if [[ -z "$run_id" ]]; then
    echo "ERROR: no airgap.yml run found for branch $branch. Run scripts/trigger_airgap_ci.sh first." >&2
    exit 2
  fi
  outdir="$(mktemp -d)"

  gh run download "$run_id" --name airgap-report --dir "$outdir"

  if [[ -z "$from_file" ]]; then
    echo "ERROR: airgap-report.json not found in downloaded artifact. Inspect $outdir manually." >&2
    exit 2
  fi
fi

echo "validating $from_file"
uv run python3 -m emmy_serve.airgap.ci_verify --from-file "$from_file"

**1. `docs/airgap-green-run.md`** — the operator runbook. This is the canonical "what a green run looks like" doc:

```markdown
# Air-Gap CI Green-Run Runbook (SC-4 Certification)

This runbook certifies Phase 1 Success Criterion 4 ("50-turn synthetic coding session
produces zero outbound network packets verified via ss/tcpdump snapshot in CI"). It is
run ONCE per self-hosted-runner lifetime (and optionally re-run on major CI changes).

## Prerequisite

The self-hosted GitHub Actions runner is registered on the DGX Spark with label
`dgx-spark`. See `docs/ci-runner.md` §1–§7. If the runner is not registered, this
runbook cannot proceed — the `airgap-replay` job will queue forever.

## Steps

### 1. Trigger the workflow

From a clean checkout on a feature branch (not `main`):

```bash
cd /path/to/emmy
git checkout -b sc4-airgap-certification
./scripts/trigger_airgap_ci.sh
```

The script:
- Verifies working tree is clean and branch is not `main`.
- Appends a timestamped marker to `air_gap/README.md` (in the workflow's path filters).
- Commits + pushes with upstream tracking.
- If `gh` CLI is installed, creates / references the PR and prints the workflow-run URL.

### 2. Observe the run

Open the PR's Checks tab (printed by `trigger_airgap_ci.sh`). Two jobs:

- `profile-hash-integrity` on `ubuntu-latest` — ~30s. Validates every
  `profiles/*/v*/` bundle via `emmy profile validate`. This job has NO GPU needs
  and is the fast gate before any self-hosted runner time is used.

- `airgap-replay` on `[self-hosted, dgx-spark]` — ~5–10 min. Boots emmy-serve with
  `--network none`, runs the D-12 four-layer probe, replays the 50-turn
  `air_gap/session.jsonl`, runs the REPRO-04 offline-HF test, and uploads
  `runs/ci-airgap/airgap-report.json` as artifact `airgap-report`.

Expected outcome: both jobs green.

### 3. Certify the run

Once `airgap-replay` completes:

```bash
./scripts/verify_airgap_ci.sh
```

The script:
- Downloads the `airgap-report` artifact via `gh run download`.
- Invokes `emmy_serve.airgap.ci_verify` to validate the JSON.
- Exits 0 iff `passes=true`, `failures=[]`, and all four D-12 layers green.

Example output (green):

```
downloading airgap-report artifact from run 12345678 -> /tmp/tmp.xyz
validating /tmp/tmp.xyz/airgap-report.json
airgap-report OK: passes=True, 4 layers green, failures=[]
  ts: 2026-04-22T03:00:00Z
  container: emmy-serve
```

Example output (red — layer (a) fail):

```
validating /tmp/tmp.xyz/airgap-report.json
airgap-report FAILED validation:
  - layer (a) network_devices not passed: expected only 'none' network, got ['bridge']
```

### 4. What counts as "4 layers green"?

Per `emmy_serve.airgap.probe`:

| Layer | Name | Proves |
|-------|------|--------|
| a | network_devices | `docker inspect` shows only the `none` network; `ip addr` inside the container shows only `lo`, no `eth*/ens*/wlan*` |
| b | dns_audit | `/etc/resolv.conf` contains no non-loopback nameservers; `getent hosts huggingface.co` fails |
| c | telemetry_env | `VLLM_NO_USAGE_STATS=1` AND `DO_NOT_TRACK=1` inside the container |
| d | hf_offline_env | `HF_HUB_OFFLINE=1` AND `TRANSFORMERS_OFFLINE=1` inside the container |

Any layer failure identifies itself in the `failures` list with its letter (a/b/c/d) —
that's the grepable audit trail.

### 5. Commit the certification evidence

Once `verify_airgap_ci.sh` exits 0, the SC-4 contract is satisfied for this branch.
Commit a certification receipt:

```bash
# Save the artifact JSON for provenance (runs/ci-airgap/ is gitignored by default
# but we commit the green-run evidence under .planning/ for durability).
mkdir -p .planning/phases/01-serving-foundation-profile-schema/evidence/
cp /tmp/airgap-report/airgap-report.json \

git add .planning/phases/01-serving-foundation-profile-schema/evidence/airgap-report-sc4-certification.json
git commit -m "$(cat <<'EOF'
chore(01-08): SC-4 certification evidence (airgap-report.json from green CI run)

- branch: <branch name>
- workflow run: <URL>
- passes: true; 4/4 layers green; failures: []
EOF
)"

git push
```

Merge the PR (or, if the marker commit from trigger_airgap_ci.sh is the only diff,
you can simply reset the branch after certification — the CI history on GitHub is
the durable record).

## Failure mode handbook

| Symptom | Likely cause | Remediation |
|---------|--------------|-------------|
| `airgap-replay` queued but never starts | self-hosted runner offline | Check runner service on DGX Spark: `systemctl status actions.runner.*` |
| Layer (a) fail: "got ['bridge']" | `start_emmy.sh --airgap` did not pass `--network none` | Check scripts/start_emmy.sh changes; confirm --airgap flag is wired to docker run |
| Layer (b) fail: "getent resolved huggingface.co" | Container has DNS reachability despite --network none (misconfigured) | Inspect container's /etc/resolv.conf; investigate host routing |
| Layer (c) fail: env var missing | serving.yaml.env regressed | `uv run emmy profile validate` locally — cross-field validator catches this normally |
| Layer (d) fail: env var missing | same as (c) | same |
| 50-turn replay errors inside the container | vLLM endpoint not ready / tool schema mismatch | `docker logs emmy-serve`; check session.jsonl unmodified from Plan 01-05 |
| `verify_airgap_ci.sh` exits 2 | `gh` CLI not installed / artifact name mismatch | Install gh OR use `--from-file` with manually-downloaded JSON |

committed scripts convert the latent risk "the CI could silently be broken" into a
deterministic two-command certification. Any future code change that breaks air-gap
discipline (e.g. adding a tool that hits network, forgetting to propagate an env var)
will fail the corresponding D-12 layer, and `verify_airgap_ci.sh` will refuse to
certify.
