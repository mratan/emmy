In `docs/runbook.md`, add a new section after the `web_search` section. Skeleton:

```markdown
## ask-claude tool — sidecar-mediated Claude Code escalation

**What it is.** A sanctioned path for Emmy's local model to ask Claude
hard reasoning questions, via the Spark's locally-installed Claude Code
CLI and OAuth (no Anthropic API key needed). Spawned by the
`emmy-sidecar` service running `claude --print -` with the prompt on
stdin.

**Default posture.** OFF. The endpoint returns 503 when the
`EMMY_ASK_CLAUDE` env var is unset on the sidecar. Operator must
explicitly flip the env to bring the surface online.

**Why this is policy-compliant.** Anthropic's 2026-02-20 Consumer Terms
update explicitly bans third-party tools using subscription OAuth
tokens directly, but explicitly supports "running the official CLI
remotely via SSH" — which is functionally equivalent to spawning
`claude --print` from another local process. The OAuth token never
leaves the Spark; only the official CLI uses it.

### Pre-flight (operator gates, do all three)

1. **Verify Anthropic Pro/Max privacy setting.** Visit claude.ai →
   Settings → Privacy → "Allow Anthropic to use your conversations
   to train future models". Toggle must be **OFF**. (For Pro/Max it
   should be off by default; verify and self-attest in
   `04.6-07-OPERATOR-PROTOCOLS.md`.)

2. **Verify `claude` CLI is on the sidecar's PATH.**
   ```sh
   systemctl --user show emmy-sidecar.service -p Environment
   # Confirm PATH includes ~/.local/bin (or wherever `claude` lives)
   command -v claude
   # Should resolve to /usr/local/bin/claude or ~/.local/bin/claude
   ```

3. **Configure the env on the sidecar unit.** Edit
   `~/.config/systemd/user/emmy-sidecar.service`:
   ```ini
   [Service]
   ...
   Environment=EMMY_ASK_CLAUDE=on
   ```
   Then `systemctl --user daemon-reload && systemctl --user restart emmy-sidecar`.

### Operator slash command (manual escalation)

```
/ask-claude What's the best Rust pattern for graceful task cancellation?
```

Routes through the sidecar; output prints in the TUI. Available in
both `pi-emmy` (Spark loopback) and `emmy` (Mac tailnet). Bypasses
the model-side `tried` gating.

### Tool form (model escalation)

Off by default per profile. To enable for the daily-driver Gemma 26B
v2.1:

```yaml
# profiles/gemma-4-26b-a4b-it/v2.1/harness.yaml
tools:
  ask_claude:
    enabled: true
    rate_limit_per_turn: 5
    rate_limit_per_hour: 30
```

Then `uv run emmy profile hash --write profiles/gemma-4-26b-a4b-it/v2.1/`
(this is an additive change per CLAUDE.md; hash-recompute in place,
no new sibling version needed).

### Privacy + threat model

| Concern | Mitigation |
|---|---|
| Prompts/code go to Anthropic | Pre-send scrubber blocks AKIA, sk-, xoxb-, private-key headers, .env paths, etc. (see `emmy_serve/tools/scrubber_config.yaml` for full list). Reject-don't-redact: model gets a structured error if the scrubber fires, can adjust and retry. |
| Anthropic data retention | Pro/Max: not used for training (per-account toggle in §pre-flight). Retained for safety review (consumer-tier floor; cannot be opted out without ZDR API tier). |
| Rate-limit abuse / runaway loops | Sidecar global cap: 30/hour, 1 in-flight, 1-call/3s min gap. Harness per-turn cap: 5/turn. Both pools shared between `pi-emmy` and `emmy`. |
| Subprocess injection | Argv list + stdin only, never shell. Prompt is data, never command. Tested. |
| OAuth credential exposure | Sidecar runs as your user (status quo); spawning `claude` is one more thing the sidecar can do. Documented; no privilege escalation. |
| Misuse — model over-uses ask_claude | Tool description requires structured `tried` arg; boilerplate `tried` ("I tried thinking") rejected. Phase 5 calibrates whether the model honors the contract. |

### Audit trail

Every `/ask-claude` call emits a JSONL event in the sidecar's
events.jsonl stream:

- `emmy.ask_claude.request` — sha256 of prompt + length + timestamp
- `emmy.ask_claude.response` — sha256 of response + length + duration_ms
- `emmy.ask_claude.error` — failure mode + structured reason
- `emmy.ask_claude.scrubber_blocked` — pattern class that fired
- `emmy.ask_claude.rate_limited` — which limit (concurrent / min_gap / hourly)

Default: NO content in the JSONL. To enable full content (debug only):
```sh
Environment=EMMY_LOG_FULL=on
```

### Kill switch

Instant disable: `Environment=EMMY_ASK_CLAUDE=off` (or unset) +
`systemctl --user restart emmy-sidecar`. Within seconds, `/ask-claude`
returns 503 and the tool form returns errors.

### Troubleshooting

- **`/ask-claude` returns 503 env_disabled** — `EMMY_ASK_CLAUDE` unset
  on the sidecar. Edit unit file, reload, restart.
- **`/ask-claude` returns 503 claude_cli_not_found** — `claude` CLI not
  on the sidecar's PATH. Install Claude Code on Spark (or fix the unit
  file's `Environment=PATH=...` to include the install dir).
- **`/ask-claude` returns 502 subprocess_failed: authentication required**
  — The Spark-side Claude Code session has expired or never been
  authenticated. SSH to the Spark, run `claude` interactively once
  to complete OAuth, then retry.
- **Scrubber blocking legit content** — Edit
  `emmy_serve/tools/scrubber_config.yaml` to refine the pattern. Restart
  the sidecar to reload.
```
