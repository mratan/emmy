**Wave:** 1 — no dependencies

**Goal:** Add an additive `tools.ask_claude` block to the harness.yaml schema (Pydantic side) and TS profile-loader. // Mirror the additive shape; match how WebSearchConfig is parsed.
```typescript
export interface AskClaudeConfig {
  enabled: boolean;
  rate_limit_per_turn: number;
  rate_limit_per_hour: number;
}

// In ToolsConfig interface:
//   ask_claude?: AskClaudeConfig;

// In the parser, with defaults:
function parseAskClaude(raw: unknown): AskClaudeConfig | undefined {
  if (raw === undefined || raw === null) return undefined;
  if (typeof raw !== "object") throw new Error("tools.ask_claude must be an object");
  const r = raw as Record<string, unknown>;
  return {
    enabled: typeof r.enabled === "boolean" ? r.enabled : false,
    rate_limit_per_turn: typeof r.rate_limit_per_turn === "number" && r.rate_limit_per_turn > 0
      ? r.rate_limit_per_turn : 5,
    rate_limit_per_hour: typeof r.rate_limit_per_hour === "number" && r.rate_limit_per_hour > 0
      ? r.rate_limit_per_hour : 30,
  };
}
```

### Task 3 — D-04 hash invariance check (LOAD-BEARING)

**This is the critical step.** After the schema change, every shipping bundle must validate **AND** its hash must be byte-identical to before. If `uv run emmy profile validate` fails because the new field changes the bundle hash, then we've violated D-04 and need to either:
(a) fix the schema to actually be additive (defaults serialize same as omission)
(b) accept new sibling versions across all 7 bundles (much heavier)

