## Decisions Made

- **Schema upper-caps beyond plan spec (`le=100` / `le=1000`)** — plan specified `gt=0` only; added upper-cap to surface accidental rate-limit foot-guns at validation time before sidecar-side enforcement. Mitigates T-02 (runaway-loop abuse) at the schema layer rather than relying solely on sidecar-side caps. Le=100 (turn) and le=1000 (hour) are generous schema upper bounds — sidecar still ships the operational ceiling (defaults to 30/hour per CONTEXT.md).
- **TS validation pattern: typed errors, not silent defaults** — chose dotted-path `ProfileLoadError` style (mirrors memory block) over silent-default style (web_search). Rationale: rate-limits affect T-02 abuse posture; silent fallback would mask operator typos. Implemented as per-field validation: absent fields use defaults; present-but-malformed throw.
- **Cast-at-use-site for ProfileSnapshot extension** — followed `web_search` precedent (snapshot carries the field but `ProfileSnapshot` type-declaration in `@emmy/provider/src/types.ts` does not declare it; sites cast as `{ ask_claude?: ... }`). Keeps the provider's narrow type contract intact; harness consumers cast. Future plan 04.6-04 (tool runtime) will codify the cast as a typed helper if needed.
- **Test directory: `tests/` over `test/`** — plan specified `packages/emmy-ux/test/profile-loader.ask-claude.test.ts` (singular `test/`); chose `packages/emmy-ux/tests/profile-loader-ask-claude.test.ts` (plural `tests/`) for byte-aligned consistency with the structurally-similar memory test (`profile-loader-memory.test.ts`). Bun test discovers both directories via `*.test.ts` glob; no functional difference.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Added schema upper-caps on rate-limit fields**
- **Found during:** Task 2 (GREEN implementation)
- **Issue:** Plan specified `Field(default=5, gt=0)` and `Field(default=30, gt=0)` only. Without an upper cap, an operator typo (`rate_limit_per_turn: 50000`) silently passes schema validation and only fails at sidecar-side enforcement (or worse, lets the model burn through the daily Pro/Max quota before hitting the per-hour ceiling). T-02 (runaway-loop / rate-limit abuse) ranks correctness-relevant per the phase threat model.
- **Fix:** Added `le=100` to `rate_limit_per_turn` and `le=1000` to `rate_limit_per_hour`. Upper bounds are generous schema sentinels — operational ceilings (30/hour) live sidecar-side per D-07. Mirrors the existing `WebSearchConfig.rate_limit_per_turn: int = Field(default=10, ge=1, le=100)` pattern.
- **Files modified:** `emmy_serve/profile/schema.py` (one line change in AskClaudeConfig); test added `test_rate_limit_per_turn_above_cap_invalid` + `test_rate_limit_per_hour_above_cap_invalid` covering the new constraint.
- **Verification:** Pytest passes; `AskClaudeConfig(rate_limit_per_turn=101)` raises `ValidationError` (was silently accepted under plan spec).
- **Committed in:** `6642035` (GREEN commit)

**2. [Rule 2 - Missing Critical] Added stricter TS validation than plan suggested**
- **Found during:** Task 2 (GREEN — TS profile-loader)
- **Issue:** Plan suggested a silent-fallback parser (`typeof r.rate_limit_per_turn === "number" && r.rate_limit_per_turn > 0 ? r.rate_limit_per_turn : 5`) — same style as the existing web_search parser. This silently coerces malformed YAML (e.g. `rate_limit_per_turn: "five"`) to defaults, masking operator typos and creating runtime divergence from the Pydantic side which throws ValidationError on the same input.
- **Fix:** Adopted the memory-block style — strict per-field validation that throws `ProfileLoadError` with dotted paths on malformed-but-present fields. Defaults still apply when fields are entirely absent (additive-schema invariant preserved). Both Pydantic and TS now reject malformed rate-limits identically; the TS test suite asserts this.
- **Files modified:** `packages/emmy-ux/src/profile-loader.ts` (~70 lines added). Includes integer + finiteness check (`Number.isInteger(v)`) since YAML numerics can be floats.
- **Verification:** TS test `invalid rate_limit_per_turn (negative) rejected with dotted-path error` passes. Pydantic and TS now agree on which payloads validate vs throw.
- **Committed in:** `6642035` (GREEN commit)

**3. [Rule 3 - Blocking] Adjusted test directory path from plan spec**
