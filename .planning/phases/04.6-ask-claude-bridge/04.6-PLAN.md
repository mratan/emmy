## Execution shape

This phase is laid out so a `gsd-executor` agent (or operator running `/gsd-execute-phase 04.6`) can pick it up cold:

1. **Each plan is self-contained.** Each `04.6-NN-PLAN.md` has its own Goal, Inputs (files to read for context), Outputs (paths to create/modify), Tasks (numbered with verification), Done Criteria (commands to run).
2. **Decisions are locked in CONTEXT.md.** Plans reference `D-XX` IDs without re-deriving rationale. If an executor encounters a decision not yet locked, the plan flags it as a Deviation (per CLAUDE.md GSD discipline) rather than guessing.
3. **Verification is binary.** Each plan ends with concrete `bash` / `bun test` / `uv run pytest` commands whose green/red exit code is the gate.
4. **Atomic commits per plan.** Each plan produces one or more commits whose messages start with `phase(04.6-NN): <action>` for traceability.

## Pre-flight (operator does this once, before execute kicks off)

Before any plan in this phase runs, operator must:

1. **Verify Anthropic Pro/Max privacy setting** (D-10). claude.ai → Settings → Privacy → "Allow Anthropic to use your conversations to train future models" toggle is **OFF**. Self-attest by writing the date + setting state into `04.6-07-OPERATOR-PROTOCOLS.md` § "Privacy attestation" (Plan 04.6-07 ships the template).
2. **Verify Claude CLI is on PATH for the sidecar process** (D-11). Spark-side: `command -v claude` returns a path; ideally `/usr/local/bin/claude` or `~/.local/bin/claude`. The sidecar `emmy-sidecar.service` user unit already extends PATH per the Phase 04.2 follow-up — confirm with `systemctl --user show emmy-sidecar.service -p Environment` and verify PATH includes `~/.local/bin`.
3. **Confirm the `EMMY_ASK_CLAUDE` env is unset** at this point. Default-off posture is the starting state.

The phase ships with all surfaces dormant; operator gestures (env flip + per-profile `enabled: true`) bring them online.
