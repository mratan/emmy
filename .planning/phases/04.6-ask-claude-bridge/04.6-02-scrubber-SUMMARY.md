## Decisions Made

- **Dataclass over Pydantic for `ScrubResult`** — it's a return type (not config), called per-prompt, value-immutable; `dataclass(frozen=True)` gives equality, hashability, and zero validator overhead. Pydantic stays where the project already uses it (profile schema, sidecar config).
- **Patterns compiled once at module import** — scrubbing is on the hot path of every `/ask-claude` request; per-call `re.compile` would be wasteful. The trade-off: changing `scrubber_config.yaml` requires sidecar restart, which is intentional per D-06's "config drift would be a bug source" comment.
- **Excerpt cap = 40 chars + ellipsis** — tested `(len(excerpt) > _EXCERPT_MAX)` branch deliberately so audit messages stay bounded. Aligns with D-08's "audit log emits sha256 fingerprints + lengths, not content" — the excerpt is the *one* exception, capped.
- **Pattern ordering preserved from YAML insertion** — Python 3.7+ dicts are ordered; `yaml.safe_load` preserves insertion order; pattern walk is deterministic. The plan's "deterministic (config-yaml ordering)" comment is honored without extra code.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Refined `jwt_or_bearer` regex middle-segment**

- **Found during:** Task 2 GREEN verification
- **Issue:** The pattern `\beyJ[a-zA-Z0-9_-]{10,}\.eyJ[a-zA-Z0-9_-]{5,}\.[a-zA-Z0-9_-]{5,}\b` from the plan required the *middle* segment to be `eyJ` followed by **at least 5** more chars. The test prompt's middle segment is just `eyJ` (3 chars total — `eyJ` then immediately a dot). Pattern under-broad → false negative on a JWT-shaped string the test contract requires us to block.
- **Fix:** Changed middle quantifier `{5,}` → `*` (zero or more). The test contract is the source of truth per the plan's "Deviations expected" note: "If a regex test case fails because the pattern is over-broad or under-broad — refine the pattern, not the test." First and third segments still bound (10+ and 5+) so we keep "JWT-shape minimum" anchoring.
- **Files modified:** `emmy_serve/tools/scrubber_config.yaml` (one line)
- **Verification:** All 20 scrubber tests green; clean-prompt parametrize unchanged (no new false positives observed on the existing clean cases).
- **Committed in:** `1d6281f` (Task 2 GREEN — fix is part of the implementation commit since the file was first introduced there)
