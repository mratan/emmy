## Decisions Made

- **`ask-claude-client.ts` as a new sibling module** (not extending `sidecar-lifecycle-client.ts`): `/ask-claude` is sync POST→JSON, not POST→SSE. Forcing it into the lifecycle client would drag in eventsource-parser unnecessarily and break Plan 04.2-04's single-responsibility split. Three sibling modules with parallel naming (status / lifecycle / ask-claude) is cleaner than a 4-shape kitchen-sink client.

- **`baseUrl` as DI argument, not env-read inside the module**: D-14 "identical UX between pi-emmy and emmy" is a property of the wire-up site (pi-emmy-extension), not this module. `ask-claude-client.ts` mirrors `sidecar-status-client.ts` — pure function of its inputs, no env coupling. The CLAUDE.md URL-config-precedence rule (env > profile > literal default) is honored at the SINGLE wire-up point, where `sidecarBaseUrl` is resolved once and passed down.

- **Reject empty / whitespace prompts client-side before round-trip**: Pydantic `min_length=1` would reject server-side too, but a round-trip for a guaranteed-rejection prompt wastes operator wait time. The `trim()` also bounds scrubber + audit-log linear cost server-side (200K-char Pydantic ceiling × cleaner inputs).

- **Plain Error + structured field bag, NOT a custom Error subclass**: Custom subclasses sometimes break `instanceof Error` checks across runtimes (V8 vs JSC; Bun's vs Node's). The TypeScript `AskClaudeError` interface declares the structured shape (`.reason`, `.pattern_class?`, `.detail?`); the runtime keeps the value as a plain `new Error(msg)` with extra fields attached. Mirrors lifecycle-client's inline-Error idiom.

- **Single-line response notify carrying response + elapsed + quota**: After a successful call the operator wants three things at a glance: the answer, how long Claude took, how many calls remain in the rolling hour budget. Putting all three in one `notify("info")` call surfaces the budget without requiring a follow-up `/status` and without splitting the rendering across multiple toast lines.

- **Description text surfaces D-05 verbatim**: `"Ask Claude (via Spark-side OAuth) a focused question — operator-invoked. Bypasses model-side rate-limits."` so when the operator types `/help` (or hovers the autocomplete), they see the exact bypass-gating contract from CONTEXT.md D-05.
