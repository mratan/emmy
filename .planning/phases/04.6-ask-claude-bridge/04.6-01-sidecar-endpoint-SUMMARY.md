## Decisions Made

- **Sentinel-default `duration_ms`** — `int | None = None` over `int = 0` with truthy check. The original plan's pseudocode used `if duration_ms:` which silently dropped a measured 0 (FakeProc returns instantly; sub-ms real spawns do too). D-08's contract is "duration_ms present on response/error events", not "present when ≥ 1ms". Auto-fix [Rule 1] inline; tested under the 9-test suite.
- **Best-effort audit-write** — `try/except` around `append_jsonl_atomic` so an unwritable JSONL parent dir never propagates to the request outcome. The atomic helper itself fsyncs and is robust; this guard exists for missing-parent-dir edge cases and operator-misconfigured EMMY_ASK_CLAUDE_JSONL paths.
- **`_audit_jsonl_path()` resolved per-call, not memoized** — long-running sidecars may have `XDG_STATE_HOME` / `EMMY_ASK_CLAUDE_JSONL` flipped without a restart; per-call resolution is one `os.environ.get` + `Path` construction (microseconds, irrelevant next to the ~30s of a real Claude turnaround).
- **`_ASK_CLAUDE_STATE` is a dict, not a dataclass** — matches Phase 04.2 module-level state precedent (`_current_profile_id`, `_vllm_pid`, `_last_error`). Test reset mutates fields in place so any captured reference stays live.
- **`asyncio.Lock` guards both the acquire AND the release** — the finally block re-takes the lock to flip `in_flight` back. Two callers can't both observe `in_flight=False` and race; the second waiter sees `True` and rejects with 429 'rate_limited_concurrent'.
- **200K-char Pydantic ceiling on prompt** — defensive bound; Anthropic Pro/Max context is well above this. Caps the linear cost of scrubber + audit log per call.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `emit_event_ask_claude.duration_ms` truthy-check dropped measured 0 ms**

- **Found during:** Task 2 GREEN verification (one test failed: `test_endpoint_emits_audit_event_request`)
- **Issue:** The plan's pseudocode declared `duration_ms: int = 0` with `if duration_ms: event["duration_ms"] = duration_ms`. The FakeProc in tests returns instantly, so `int((time.monotonic() - t0) * 1000) == 0`. The truthy guard then dropped the field. D-08 says events carry "duration_ms" on response/error — silently omitting it on a 0-ms operation is a data-quality bug (not a privacy bug, but a contract-shape bug for downstream consumers).
- **Fix:** Changed signature to `duration_ms: int | None = None` and the guard to `if duration_ms is not None: event["duration_ms"] = int(duration_ms)`. Now a caller-explicit 0 records correctly; only callers that don't intend to log timing (e.g. `request` event before the spawn) leave the field absent.
- **Files modified:** `emmy_serve/swap/controller.py` (one helper signature + one guard line)
- **Verification:** All 9 ask-claude tests pass; 296 total unit tests pass (zero regressions on the existing 287).
- **Committed in:** `a1ccccd` (Task 2 GREEN — fix is part of the implementation commit since the helper was first introduced there)

## User Setup Required

None — the endpoint is purely additive code. Operator-side activation requires:

- `EMMY_ASK_CLAUDE=on` set on the sidecar process (D-03; default is OFF / 503).
- `claude` binary on the sidecar process's PATH (D-11; the user-systemd unit's PATH-extending env file from Phase 04.2 follow-up already covers this on operator hardware).
- (Optional) `EMMY_ASK_CLAUDE_JSONL=<path>` to override the audit JSONL location; defaults to `${XDG_STATE_HOME:-~/.local/state}/emmy/sidecar/ask_claude_events.jsonl`.
- (Optional, debug only) `EMMY_LOG_FULL=on` to capture raw prompt/response in audit events; default OFF preserves D-08's "fingerprint-only" privacy posture.

These are documented in upcoming Plans 04.6-06 (docs) and 04.6-07 (operator protocols); this plan ships only the code.
