# Phase 04.6 — Ask-Claude bridge: Spark-side Claude Code OAuth via sidecar

**Status:** SCOPED — ready to plan/execute
**Drafted:** 2026-04-29 (late evening)
**Targeted execute window:** post-system-reboot 2026-04-30 (after GPU is back)
**Phase shape:** mid-milestone insertion, extension of Phase 04.2 sidecar surface

## Purpose

Give the local Emmy model a sanctioned path to ask Claude for help on hard reasoning problems, without violating Anthropic's subscription-OAuth ToS and without committing to a paid Anthropic API key. The capability is hosted on the Spark (where `emmy-serve` and `emmy-sidecar.service` already live), exposed as one more sidecar endpoint, and consumed identically by `pi-emmy` (Spark loopback) and `emmy` (Mac tailnet) harness modes.

## Background

- Anthropic ToS update **2026-02-20** + enforcement **2026-04-04** explicitly bans third-party tools using subscription OAuth tokens directly. Subscription OAuth is sanctioned **only** for Claude.ai web + the official Claude Code CLI.
- Same policy explicitly supports **"running the official CLI remotely via SSH"**, which is functionally equivalent to "any process invoking `claude --print` on the machine that owns the OAuth token."
- Operator runs Claude Code with OAuth on **both** the Mac and the Spark. This phase routes all Emmy-driven escalations through the **Spark**'s Claude Code install — single chokepoint, single audit trail, single credential location, mirrors the existing "Spark hosts capabilities; harnesses are clients" model.
- Anthropic Pro/Max consumer tier: prompts/outputs **not used for training** by default (operator must verify in claude.ai → Settings → Privacy → "Help improve Claude" toggle off). Retention for safety/abuse review is the floor and cannot be opted out at this tier; for our occasional-escalation use case this is acceptable.

See discussion thread in this session's transcript (operator + claude, 2026-04-29) for the full architectural deliberation.

## Locked Decisions

| ID | Decision | Rationale |
|----|---|---|
| **D-01** | **Spark-side OAuth, single chokepoint.** Sidecar on Spark spawns `claude --print` using Spark's OAuth. Mac's Claude Code OAuth is not used by Emmy. | Single audit surface, single credential location, mirrors emmy-serve hosting model. Mac-side OAuth stays untouched. |
| **D-02** | **Extend the existing Phase 04.2 sidecar with `POST /ask-claude`.** Do not introduce new infra. | The sidecar is already the "Spark capability surface" (Phase 04.2 D-04 BYTE-STABLE). Adding `/ask-claude` is one more endpoint with the same auth model (loopback / tailnet-restricted) and the same observability shape (`emmy.swap.*` JSONL → `emmy.ask_claude.*`). |
| **D-03** | **Default OFF via `EMMY_ASK_CLAUDE=on` env on the sidecar process.** Endpoint returns 503 when the env is unset, even before any rate-limit / scrubber checks. | Air-gap default preserved. Mirrors `EMMY_WEB_SEARCH=off` posture (Phase 3.1 / D-33 LOCKED). Operator gesture required to bring the surface online. |
| **D-04** | **Profile change is strictly additive.** New top-level `tools.ask_claude` block whose absence validates as `None`; recompute bundle hash in place per CLAUDE.md immutability rule. No new sibling version cut. | Same exemption Phase 04.4-03 took for `context.memory:` and Phase 04.5-02 for `subagents:`. Behavioral byte-set is unchanged when `enabled: false` (default). |
| **D-05** | **Two invocation paths with different gates.** Slash command `/ask-claude <q>` is operator-only, no model-side gating, no per-turn rate-limit. Tool form `ask_claude(question, tried, relevant_context)` is opt-in per profile, requires structured args, enforces per-turn rate-limit. Both route to the same sidecar endpoint. | Slash = explicit human ask = trust. Tool = automated model decision = needs gating to prevent over-use. |
| **D-06** | **Pre-send scrubber on the sidecar side.** Operator-configurable regex blocklist + glob blocklist (e.g., `AKIA[A-Z0-9]{16}`, `sk-[a-zA-Z0-9]{32,}`, `.env*`, `*.key`, `secrets/**`). Scrubber rejects the call (returns structured error) rather than silently redacting, so the model sees the failure and can adjust. | Privacy first; explicit failure beats silent corruption. Reject-don't-redact mirrors how `web_fetch` handles allowlist failures. |
| **D-07** | **Rate limits split between sidecar (global) and harness (per-turn).** Sidecar enforces 30 calls/hour + 1 in-flight + 1-call-per-3s minimum gap. Harness enforces 5 calls/turn (configurable per profile). Pools shared across `pi-emmy` and `emmy` modes. | Single-source-of-truth on global pool; harness-side per-turn limit shapes model behavior locally. |
| **D-08** | **Audit log emits sha256 fingerprints + lengths + timing, not content.** `emmy.ask_claude.{request,response,error,rate_limited}` events include `prompt_sha256`, `response_sha256`, `prompt_len`, `response_len`, `duration_ms`, `rate_limit_state`. Full content logged only when `EMMY_LOG_FULL=on` (debug). | Default privacy posture: cannot reconstruct prompt/response from the JSONL stream. Full-content logging is operator gesture only. |
| **D-09** | **Subprocess invocation via argv + stdin, never shell.** Sidecar spawns `["claude", "--print", "-"]` and pipes the prompt to stdin. Prompt is **data**, never command-line. | Closes the command-injection surface. SSH equivalent: when SSH is the transport, `ssh user@host '/usr/local/bin/claude --print -'` with `command="..."` forced-command in `authorized_keys`. |
| **D-10** | **Operator pre-flight requires verifying Anthropic Pro/Max privacy setting.** claude.ai → Settings → Privacy → "Help improve Claude / Allow Anthropic to use your conversations to train future models" toggle must be **OFF**. Phase pre-flight script asserts this is documented in `OPERATOR-PROTOCOLS.md`. | We can't programmatically verify the setting from outside Anthropic's UI; this is a documented operator gate, similar to Phase 04.4 operator-gated SC items. |
| **D-11** | **`claude` CLI must be on the sidecar process's PATH on Spark.** Sidecar pre-flight checks `command -v claude` at startup; if absent, the `/ask-claude` endpoint returns 503 with `"reason": "claude_cli_not_found"`. | Fail-closed when prerequisites aren't met. The user-systemd unit already extends PATH per the Phase 04.2 follow-up; we just need to confirm `claude` is reachable. |
| **D-12** | **Sidecar runs as user that owns OAuth.** Status quo per Phase 04.2 unit file (`%h` paths). Documented as an explicit privilege boundary: "sidecar can spawn `claude` and therefore can use your OAuth subscription." | No new privilege escalation. The sidecar already has access to your home dir; spawning `claude` is one more thing it can do. Documenting it makes the threat model explicit. |
| **D-13** | **Tool form opt-in per profile via `tools.ask_claude.enabled: true`.** Operator flips the switch on the bundles where they want model-side escalation. Default false on every bundle. | Profile-by-profile control. Daily-driver Gemma 26B v2.1 might enable it; eval-only audit profiles might not. |
| **D-14** | **`pi-emmy` and `emmy` harnesses share identical UX.** Slash command + tool form behave identically regardless of whether the harness runs on the Spark (loopback to `127.0.0.1:8003/ask-claude`) or the Mac (tailnet to `https://<spark>:8003/ask-claude`). | Single mental model for the operator; the dispatcher already routes through `EMMY_SERVE_URL` per Phase 04.2 D-33. |
| **D-15** | **"X circumstances" guidance is shipped in the tool description, not enforced in code.** Phase 5 eval calibrates whether the model uses `ask_claude` appropriately. The tool itself only enforces the structured-args gate (`tried` field non-trivial). | Discipline shaping via prompt + observability, not via hard rules. Same approach as memory tool (V1/V2/V3 calibration in V-RESULTS-v1..v8). |

## Threat Model

| ID | Threat | Mitigation |
|----|---|---|
| **T-01** | **Data exfiltration via prompts.** Model accidentally puts secrets, PII, or proprietary code in `ask_claude` prompts. Anthropic retains for safety review even on Pro/Max. | Pre-send scrubber (D-06) blocks obvious patterns. Tool description explicitly forbids "secrets, credentials, PII, customer data." Structured `relevant_context` arg encourages bounded context (≤ 2K tokens). Per-turn rate-limit (D-07) caps blast radius. |
| **T-02** | **Rate-limit abuse / runaway loops.** Bug or prompt-injection causes the model to call `ask_claude` 100× per turn, burning Pro/Max rate limit and tripping abuse detection. | Sidecar global limit + harness per-turn limit (D-07). Both enforced before subprocess spawn. Hard ceiling: 30/hour. Operator can flip `EMMY_ASK_CLAUDE=off` to kill instantly. |
| **T-03** | **Subprocess injection.** Prompt contains `claude --print -` interactive escapes or shell metacharacters that break out of the spawned process. | Argv list + stdin, never shell (D-09). Prompt is data only. Tested by integration test that includes shell-metacharacter prompts. |
| **T-04** | **Audit gap.** A call happens but no JSONL event is recorded; we cannot reconstruct what the model asked Claude after the fact. | Every `/ask-claude` invocation emits at least one JSONL event (`request`, then `response` or `error` or `rate_limited`). Sidecar's atomic JSONL append (`emmy_serve/diagnostics/atomic.py` per Phase 04.4-06) ensures the event is fsync'd before subprocess return. |
| **T-05** | **OAuth credential exposure.** If Spark is compromised, attacker can shell out to `claude --print` using operator's subscription. | Documented (D-12). The sidecar already has access to operator's home dir, so this is incremental, not net-new. Mitigation: ship `EMMY_ASK_CLAUDE=on` only on the operator's own Spark (not on shared/CI machines). Anthropic's abuse detection covers the rest. |
| **T-06** | **Misuse — model over-uses `ask_claude`.** Model treats it as a confidence signal and calls Claude on trivial questions, defeating the local-first thesis. | Tool description with explicit "X circumstances" (D-15). Structured `tried` arg forces the model to document prior attempts (tool returns error if `tried` is empty / boilerplate). Phase 5 eval-calibrates with V-protocol-style probes; iterate on description if usage is wrong. |
| **T-07** | **Privacy regression.** Operator forgets to disable Anthropic training-data sharing; their code ends up training a future Claude model. | Documented operator pre-flight gate (D-10). Phase pre-flight script (Plan 04.6-07) lists the explicit checkbox the operator must verify before flipping `EMMY_ASK_CLAUDE=on`. |

## Deferred / Out of Scope

- **Multi-provider escalation** — only Claude Code via official CLI. Bedrock/Vertex/OpenAI/Gemini deferred to a future phase if needed.
- **Streaming responses** — sync request/response only in v1. Streaming requires SSE plumbing in the sidecar similar to `/start`'s D-02 progress phases; nice-to-have but not essential for occasional-escalation use case.
- **Cost tracking / token accounting** — Pro/Max is flat-rate; not needed.
- **Mac-side fallback path** — D-01 locks Spark-side OAuth. If Spark is unreachable, `ask_claude` is unavailable; operator can flip on Mac-side as a follow-up if needed.
- **Eval calibration of "X circumstances"** — Phase 5 follow-up. This phase ships the contract; Phase 5 measures whether the model honors it.
- **Multi-question batching / consultation history** — single-shot Q+A only. No "follow-up to my last ask_claude question" support.
- **Engine-swap "consult heavy local"** — separate phase if/when Mistral M3.5 4-bit or Qwen 3-Coder 80B lands as a profile.

## Acceptance Gates

The phase closes when **all** of the following pass:

1. **Sidecar endpoint integration test green** — mock `claude` binary returns canned response; scrubber blocks dirty prompts; rate-limit blocks 31st call in an hour; audit JSONL captures all four event types.
2. **Air-gap CI fail-closed regression** — with `EMMY_ASK_CLAUDE` unset, `/ask-claude` returns 503; the existing `emmy_serve.airgap.ci_verify_phase3` STRICT validator continues to pass (no new outbound surface visible from Emmy's harness when the env is off).
3. **`pi-emmy` slash command live walkthrough** — operator runs `/ask-claude "what's 2+2"` from a Spark-side `pi-emmy` session; gets answer; JSONL event captured; resume signal `ask-claude pi-emmy green`.
4. **`emmy` slash command live walkthrough** — same flow from a Mac-side `emmy` session over Tailscale; resume signal `ask-claude emmy green`.
5. **Tool form discipline check** — operator-monitored multi-turn coding session with the tool form enabled; verify the model uses `ask_claude` ≤ 5 times in a 30-minute session and only on hard reasoning problems (not routine edits / lookups). Resume signal `ask-claude discipline green`.
6. **Scrubber blocking test** — operator manually attempts `/ask-claude "my OPENAI_API_KEY=sk-fake1234..."`; sidecar rejects with structured error; resume signal `ask-claude scrubber green`.
7. **Rate-limit test** — operator scripts 32 calls; first 30 succeed, 31st blocks with `rate_limited` event; resume signal `ask-claude ratelimit green`.
8. **Privacy pre-flight documented** — `OPERATOR-PROTOCOLS.md` explicit checkbox: "Anthropic training-data sharing toggled OFF in claude.ai settings." Operator self-attests before flipping `EMMY_ASK_CLAUDE=on`.
