## SC-5 — Rate-limit test

**Resume-signal:** `ask-claude ratelimit green`

1. Script 32 sequential calls over ~5 minutes (slow enough to bypass min-gap):
   ```sh
   for i in {1..32}; do
     sleep 4   # bypass 3s min-gap
     curl -s -X POST http://127.0.0.1:8003/ask-claude \
         -H 'content-type: application/json' \
         -d "{\"prompt\":\"call $i\"}" | jq -c .
   done
   ```
2. Confirm calls 1-30 return 200; call 31 returns 429 with `reason: "rate_limited_hourly"`.
3. JSONL stream captured 30 `request`/`response` pairs and 2 `rate_limited` events.
4. Wait 1 hour, run one more call — confirm it succeeds (window slid).
5. - [ ] Walkthrough green; signing `ask-claude ratelimit green` with date ____.

## SC-6 — Air-gap fail-closed regression

**Resume-signal:** `ask-claude airgap green`

1. Edit unit file, set `EMMY_ASK_CLAUDE=off` (or remove it), reload + restart sidecar.
2. `/ask-claude what is 2+2` from `pi-emmy` → TUI shows error referencing `EMMY_ASK_CLAUDE`.
3. `curl -X POST http://127.0.0.1:8003/ask-claude` → 503 `env_disabled`.
4. Run `uv run python -m emmy_serve.airgap.ci_verify_phase3 --dry-run` (the STRICT validator). Exit code 0 — no regression in air-gap CI surface.
5. Run `uv run python -m emmy_serve.airgap.ci_verify_research_egress --dry-run` (the PERMISSIVE validator). Exit code 0.
6. Re-flip env back to `on` for ongoing use.
7. - [ ] Walkthrough green; signing `ask-claude airgap green` with date ____.
