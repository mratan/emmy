## Summary

**Top recommendations (one per open question):**

1. **Sidecar↔orchestrator IPC: subprocess.** Reuse the `subprocess.Popen("uv", ["run", "python", "-m", "emmy_serve.swap.orchestrator", ...])` invocation today's TS path uses verbatim. The orchestrator is CLI-shaped (argparse `main()`; it `print(json.dumps(rec), flush=True)` to stdout; on failure it `return 5`/`return 6` from `swap_profile()`). Module-import would require refactoring `swap_profile()` to yield a generator instead of `print()`-ing — Phase-4 D-02 progress contract change. Subprocess preserves the stdout-line-buffered contract that's already locked. Cost: one `uv run` startup hit per swap (~1–2 s), trivial vs the multi-minute swap itself.

2. **FastAPI SSE: use `sse-starlette`'s `EventSourceResponse` (>=3.0.0)** rather than hand-rolling `StreamingResponse(media_type="text/event-stream")`. Built-in keepalive ping (`ping=15` default), `request.is_disconnected()` polling baked in, `shutdown_grace_period` for clean cancel-on-disconnect. CONTEXT D-06 says "SSE via FastAPI's StreamingResponse" but the discretion text allows the planner to pick the exact mechanism — sse-starlette is a 1-line dep that eliminates 30 LOC of hand-rolled keepalive + disconnect-detect.

3. **systemd user unit: `Type=simple`, `ExecStart=%h/.local/bin/uv run python -m emmy_serve.swap.controller`**, `Restart=on-failure`, `RestartSec=5`, `WantedBy=default.target`. `loginctl enable-linger $USER` documented as a one-time op step. `Environment=PYTHONUNBUFFERED=1` is mandatory (line-buffered stdout = lost progress events).

4. **Tailscale Serve multi-route: separate ports, three `--bg` invocations** (`8002` vLLM, `8003` sidecar, `8888` SearxNG). `--bg` configs persist across reboot per official docs. Path-prefix routing (`--set-path`) works but adds a base-URL transform layer in TS code that doesn't pull weight here. Separate-port keeps the local-mode URLs (`http://127.0.0.1:8003`) byte-identical to the remote-mode URLs (`https://spark.tailnet.ts.net:8003`) modulo scheme + host — minimal config divergence.

5. **Bun SSE consumer: hand-rolled with `eventsource-parser` (npm, ~3KB)** behind a `fetchSseImpl` DI seam mirroring today's `spawnFn` pattern. Bun's `fetch` returns a real `Response.body: ReadableStream`; pipe through `createParser()` from `eventsource-parser`, dispatch on `data:` events, abort via `AbortController` on Ctrl+C. Avoid the browser `EventSource` polyfill — it doesn't support `AbortController`/POST and we need both.

6. **Dual-path dispatcher: `if (process.env.EMMY_REMOTE_CLIENT === "1") return runViaHttp(args); return runViaSpawn(args);`** as the very first branch in `runSwapAndStreamProgress`. Local path stays byte-identical (snapshot test on the spawn argv). HTTP path lives in a sibling export `runSwapAndStreamProgressHttp` with the same signature minus `spawnFn`, plus a `fetchSseImpl` test seam.

7. **vLLM `/metrics` parse: use `prometheus_client.parser.text_string_to_metric_families`** (already a dependency at `prometheus-client>=0.20`; current 0.24.1 — pyproject pin satisfied). 5 LOC vs porting the TS regex parser. No new dep needed.

8. **State machine: hand-rolled enum + `dict[State, set[State]]` transition table** (~30 LOC). Both `transitions` (0.9.3) and `python-statemachine` (3.0.0) are well-maintained but their callback model is overkill for 6 states + ~7 edges. Hand-rolled is testable, inspectable, and stamps trivially into the `/status` payload's `state` field.

**Primary recommendation:** Start the sidecar as a 6-endpoint FastAPI app (`POST /start`, `POST /stop`, `GET /status`, `POST /profile/swap`, plus an internal `GET /healthz` and `GET /` for sanity). Subprocess to existing `swap.orchestrator`, capture stdout, re-emit lines as SSE through `sse-starlette` `EventSourceResponse`. systemd user unit owns lifecycle. Three Tailscale Serve `--bg` routes (8002, 8003, 8888) provisioned by `install-client.sh`. TS dispatcher gets a single env-flag branch with a snapshot-tested local hot path.

## Resolved Decisions

### 1. Sidecar↔orchestrator IPC — **subprocess.Popen** (not module import)

**Chosen:** `subprocess.Popen("uv", ["run", "python", "-m", "emmy_serve.swap.orchestrator", "--from", X, "--to", Y, "--port", "8002"], stdout=PIPE, stderr=PIPE, bufsize=1, text=True)` then `for line in proc.stdout: yield json.loads(line)`.

**Citation:** `[VERIFIED: emmy_serve/swap/orchestrator.py:90-150]` — orchestrator emits via `progress.emit()` which `print(json.dumps(rec), flush=True)`. The `swap_profile()` function returns an `int` exit code, not a generator. Refactoring it to yield would break the Phase-4 D-02 LOCKED stdout contract that the existing TS dispatcher (`profile-swap-runner.ts:107-141`) already parses.

**Alternatives considered:**

| Option | Why rejected |
|--------|--------------|
| `import emmy_serve.swap.orchestrator; orchestrator.swap_profile(...)` | Would force `swap_profile` to yield (breaks Phase-4 D-02 contract); also blocks the FastAPI event loop unless wrapped in `run_in_executor`. Re-raising the JSON-per-line as Python events is more code than just reading stdout. |
| Refactor orchestrator to expose `def run_swap_async() -> AsyncIterator[ProgressEvent]` | Phase-4 closed Done† on operator-gated thermal evidence; refactoring the primitive forces a re-validation we explicitly don't want (CONTEXT.md D-04 rationale). |

**Trade-offs accepted:** ~1–2 s `uv run` startup cost per swap (negligible vs ~3–8 min total swap time); subprocess management overhead (signal forwarding, cleanup on cancel) — but already covered by D-01's drain logic.

### 2. FastAPI SSE library — **sse-starlette `EventSourceResponse`** (3.3.4 current)

**Chosen:** `from sse_starlette.sse import EventSourceResponse` returning `EventSourceResponse(event_generator(request), ping=15, send_timeout=10, shutdown_grace_period=2)`.

**Citation:** `[VERIFIED: github.com/sysid/sse-starlette]` — the library handles four things hand-rolled SSE gets wrong:
1. Periodic `: keep-alive\n\n` pings on a `ping=N` interval (default 15 s — well under Tailscale Serve's idle timeout).
2. Async client-disconnect detection via `request.is_disconnected()` polled in the response loop.
3. Cooperative cancel on server shutdown (`shutdown_event` + `shutdown_grace_period`).
4. Correct `Cache-Control: no-cache` + `X-Accel-Buffering: no` headers (the latter matters for any reverse-proxy hop, including Tailscale Serve).

`[CITED: sse-starlette PyPI 3.3.4 release Apr 2026]` — actively maintained, MIT, ~250 LOC.

**Alternatives considered:**

| Option | Why rejected |
|--------|--------------|
| Hand-rolled `StreamingResponse(media_type="text/event-stream")` | ~30 LOC for proper keepalive + disconnect detection that sse-starlette already ships. Easy to forget `flush` semantics, easy to leak the subprocess on client disconnect. |
| `fastapi-sse` (community fork) | Less actively maintained, no shutdown-grace support. |
| WebSockets | Overkill for one-way push; D-03 already locked SSE shape. |

**Note vs CONTEXT D-06:** D-06 says "SSE via FastAPI's StreamingResponse(media_type='text/event-stream')". sse-starlette's `EventSourceResponse` is a `StreamingResponse` subclass that adds the four behaviors above — the planner can treat this as a strict superset of D-06. If the planner prefers strict literal D-06 compliance, the hand-roll is feasible but recommend documenting the keepalive cadence + disconnect-poll loop explicitly.

### 3. systemd user unit ExecStart — **`uv run python -m emmy_serve.swap.controller`**

**Chosen:** Service file at `emmy_serve/systemd/emmy-sidecar.service` (in-repo, copied to `~/.config/systemd/user/` by `start_emmy.sh` install step):

```ini
[Unit]
Description=Emmy sidecar — vLLM lifecycle controller (Phase 04.2)
Documentation=file:///%h/code/emmy/docs/runbook.md
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=%h/code/emmy
ExecStart=%h/.local/bin/uv run python -m emmy_serve.swap.controller
Restart=on-failure
RestartSec=5
TimeoutStopSec=40
KillSignal=SIGTERM
Environment=PYTHONUNBUFFERED=1
Environment=VLLM_NO_USAGE_STATS=1
Environment=VLLM_LOAD_FORMAT=fastsafetensors
Environment=HF_HOME=%h/.cache/huggingface
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=default.target
```

**Why these values:**

- **`Type=simple`** — sidecar doesn't fork; uvicorn starts immediately and binds. `Type=notify` would require integrating sd_notify which is overkill for a single-user box.
- **`%h/.local/bin/uv run`** — uv is the project's installed package manager (start_emmy.sh requires it). `%h` is the systemd specifier for `$HOME` so the unit is portable. Direct `uv` rather than `python` because uv handles venv resolution and dep sync — matches today's TS-side spawn pattern (`runSwapAndStreamProgress` invokes `"uv"`).
- **`Restart=on-failure` + `RestartSec=5`** — survive transient crashes; 5 s back-off prevents a thrash loop. Don't use `Restart=always` because that would mask intentional shutdowns (e.g. operator running `systemctl --user stop` for maintenance).
- **`TimeoutStopSec=40`** — must exceed D-01's 30 s drain + 5 s SIGKILL window with a small margin. Without this, systemd will SIGKILL the sidecar mid-drain.
- **`PYTHONUNBUFFERED=1`** — line-buffered stdout is mandatory (mirrors the TS-side parsing assumption from `progress.py:emit`).
- **`StandardOutput=journal`** — sidecar logs reachable via `journalctl --user -u emmy-sidecar -f` (single source of truth, structured time stamps).
- **`After=network-online.target` + `Wants=network-online.target`** — `tailscale serve` and the loopback bind both need network, but uvicorn binds explicitly so `network.target` is sufficient — `network-online.target` is belt-and-braces.

**One-time op step:** `loginctl enable-linger $USER` (so the user manager survives logout). Documented in runbook + install-client.sh.

`[CITED: ArchWiki systemd/User]` (verified 2026-04-25) + `[CITED: oneuptime.com 2026-03-02 systemd user services Ubuntu guide]`.

**Alternatives considered:**

| Option | Why rejected |
|--------|--------------|
| `ExecStart=/usr/bin/python3 -m emmy_serve.swap.controller` | Skips uv → wrong PYTHONPATH (no project venv); deps not resolved. |
| `ExecStart=%h/.venv/bin/python -m emmy_serve.swap.controller` | Hardcodes a specific venv path; uv may put it elsewhere; doesn't auto-sync if pyproject changes. |
| `Type=notify` with sd_notify integration | Overkill. Single-user box; restart-on-failure is acceptable cold-start UX. |
| Docker container | Adds Docker-in-Docker complexity for spawning vLLM container; CONTEXT D-05 explicitly rejects this. |

### 4. Tailscale Serve multi-route — **three separate `--bg` invocations on three ports**

**Chosen:** On Spark (run once during install or in `start_emmy.sh`):

```bash
tailscale serve --bg --https=8002 http://127.0.0.1:8002    # vLLM (existing)
tailscale serve --bg --https=8003 http://127.0.0.1:8003    # sidecar (new)
tailscale serve --bg --https=8888 http://127.0.0.1:8888    # SearxNG (new)
```

`[VERIFIED: tailscale.com/docs/reference/tailscale-cli/serve]` — `--bg` configurations persist across reboots automatically. Each invocation registers an independent serve config; multiple ports coexist on a single Tailscale node (the `tailscale serve status` lists all).

**On Mac client (written by `install-client.sh` into the wrapper script):**

```sh
EMMY_REMOTE_CLIENT=1 \
EMMY_SERVE_URL="https://${EMMY_SPARK_HOST}:8003" \
EMMY_SEARXNG_URL="https://${EMMY_SPARK_HOST}:8888" \
... existing wrapper invocation ...
```

Note: the existing wrapper already passes `--base-url https://${EMMY_SPARK_HOST}` to pi-emmy (which routes to vLLM :8002 — actually vLLM Tailscale Serve route is on 443/the default HTTPS port today per `install-client.sh:150` — verify with operator if the new sidecar should reuse 443 with a path prefix or take a separate port). **Recommendation: separate ports** — keeps URL math identical local↔remote (`http://127.0.0.1:8003` ↔ `https://spark.tailnet.ts.net:8003`).

**Alternatives considered:**

| Option | Why rejected |
|--------|--------------|
| Single hostname + `--set-path` (`/sidecar/`, `/searxng/`) | Adds path-prefix logic to TS code (need to URL-join when constructing requests). Local mode has no path prefix → divergent code paths. |
| Separate Tailscale node for sidecar | Kills the "one Spark" mental model; auth + routing complexity. |
| Tailscale Funnel (public internet) | Wrong layer — single-user single-tailnet posture (C-05). |

**Idle-timeout note:** Tailscale Serve uses HTTPS termination but does NOT have a documented idle timeout for SSE streams (no public spec). The 15 s `ping` keepalive from sse-starlette is the defensive measure. If users report SSE drops mid-swap, lower `ping` to 10 s.

### 5. Bun SSE consumer — **hand-rolled with `eventsource-parser` behind a DI seam**

**Chosen:** New file `packages/emmy-ux/src/profile-swap-runner-http.ts` exporting `runSwapAndStreamProgressHttp({from, to, port, onProgress, fetchSseImpl?})`. Implementation:

```ts
import { createParser } from "eventsource-parser";  // ~3 KB, MIT, actively maintained

export type FetchSseImpl = (
  url: string,
  init: RequestInit & { signal: AbortSignal },
) => Promise<Response>;

export async function runSwapAndStreamProgressHttp(args: {
  from: string;
  to: string;
  port: number;
  onProgress: (phase: string, pct?: number) => void;
  fetchSseImpl?: FetchSseImpl;
}): Promise<SwapResult> {
  const fetchSse: FetchSseImpl = args.fetchSseImpl ?? ((url, init) => fetch(url, init));
  const ctl = new AbortController();
  const url = `${process.env.EMMY_SERVE_URL ?? "http://127.0.0.1:8003"}/profile/swap`;
  const resp = await fetchSse(url, {
    method: "POST",
    headers: { "content-type": "application/json", accept: "text/event-stream" },
    body: JSON.stringify({ from: args.from, to: args.to, port: args.port }),
    signal: ctl.signal,
  });
  if (!resp.ok || !resp.body) {
    return { exit: resp.status >= 500 ? 1 : 1 };
  }
  let envelope: SwapResult["envelope"];
  let exitCode = 0;
  const parser = createParser((event) => {
    if (event.type !== "event") return;
    try {
      const rec = JSON.parse(event.data);
      if (typeof rec.phase === "string") {
        args.onProgress(rec.phase, typeof rec.pct === "number" ? rec.pct : undefined);
      } else if ("rolled_back" in rec) {
        envelope = { rolled_back: rec.rolled_back, rollback_succeeded: rec.rollback_succeeded };
      } else if ("exit" in rec) {
        exitCode = rec.exit;
      }
    } catch { /* ignore malformed */ }
  });
  const decoder = new TextDecoder();
  for await (const chunk of resp.body as unknown as AsyncIterable<Uint8Array>) {
    parser.feed(decoder.decode(chunk, { stream: true }));
  }
  return { exit: exitCode, envelope };
}
```

**Citation:** `[VERIFIED: bun.com/docs/guides/http/sse]` — Bun supports `for await (const chunk of resp.body)` directly. `[VERIFIED: npm view eventsource-parser version → 3.0.x as of 2026]`. `[CITED: lukeed/fetch-event-stream]` is a smaller alternative (741 bytes) but eventsource-parser has better TS types and is the de facto standard.

**Why `eventsource-parser` not browser `EventSource`:** The browser `EventSource` is GET-only and doesn't accept request bodies — but `/profile/swap` is POST (request body carries `{from, to, port}`). Also `EventSource` doesn't accept an `AbortController` signal — needed for Ctrl+C cancel.

**DI seam mirrors today's `spawnFn` pattern** (profile-swap-runner.ts:78). Tests inject a fake `fetchSseImpl` that yields a fake Response with a controlled body.

**Alternatives considered:**

| Option | Why rejected |
|--------|--------------|
| Browser `EventSource` polyfill | GET-only; no AbortController. |
| `fetch-event-stream` (lukeed) | Smaller (741 B) but no TS types; eventsource-parser is the well-known standard. |
| Hand-rolled `data: ` line splitter | ~40 LOC; trivially wrong edge cases (multi-line `data:`, `event:`, `id:`, `retry:` fields per SSE spec). Use the 3 KB lib. |

### 6. Dual-path dispatcher — **leading env branch + sibling export**

**Chosen:** Top of `runSwapAndStreamProgress`:

```ts
export async function runSwapAndStreamProgress(args: {...}): Promise<SwapResult> {
  // EMMY_REMOTE_CLIENT=1 → HTTP path; anything else → byte-stable spawn path.
  if (process.env.EMMY_REMOTE_CLIENT === "1") {
    const { runSwapAndStreamProgressHttp } = await import("./profile-swap-runner-http");
    return runSwapAndStreamProgressHttp(args);
  }
  // ===== EXISTING SPAWN PATH (UNCHANGED below this line) =====
  ...
}
```

**Test strategy that locks the local path at byte equivalence:**

1. **Argv snapshot test** — assert that with `EMMY_REMOTE_CLIENT` unset, the `spawnFn` is invoked with EXACTLY this argv array (regression guard against accidental refactors):
   ```ts
   ["uv", ["run", "python", "-m", "emmy_serve.swap.orchestrator", "--from", "X", "--to", "Y", "--port", "8002"], {cwd: ..., stdio: ["ignore", "pipe", "inherit"]}]
   ```
2. **Branch-coverage test** — `EMMY_REMOTE_CLIENT=1` triggers HTTP path; `EMMY_REMOTE_CLIENT=0` and unset both trigger spawn path.
3. **Hot path is BYTE-stable** — no new `try`/`catch`, no new `await` in the spawn branch. The lazy `await import()` of the HTTP module ensures HTTP mode dependencies aren't loaded in local mode (zero overhead).

**Why "leading env branch + lazy import" not "two separate exports":** The CONTEXT C-06 + D-04 contract says `harness-swap.ts`, the `/profile` slash command, and `pi-emmy-extension.ts` (which constructs the slash command with `runSwap: runSwapImpl`) ALL pass through a single `runSwapAndStreamProgress` symbol. Changing the symbol would cascade. Single entry, branched internally, lazy-loaded HTTP module = zero local-mode regression risk.

### 7. vLLM `/metrics` polling from Python — **`prometheus_client.parser.text_string_to_metric_families`**

**Chosen:** New module `emmy_serve/swap/vllm_metrics.py`:

```python
import httpx
from prometheus_client.parser import text_string_to_metric_families

async def fetch_vllm_metrics(base_url: str, timeout_s: float = 2.0) -> dict[str, float]:
    async with httpx.AsyncClient(timeout=timeout_s) as client:
        r = await client.get(f"{base_url.rstrip('/')}/metrics")
        r.raise_for_status()
    out: dict[str, float] = {}
    for family in text_string_to_metric_families(r.text):
        for sample in family.samples:
            out[sample.name] = sample.value
    return out
```

`vllm:gpu_cache_usage_perc` (Gauge, 0-1) is the KV-used-pct source verified at `[CITED: docs.vllm.ai/en/stable/usage/metrics/]`.

**Citation:** `[VERIFIED: prometheus_client.parser]` — already in `pyproject.toml [project.dependencies]` as `prometheus-client>=0.20` (current 0.24.1 installed on Spark). Zero new deps.

**Alternatives considered:**

| Option | Why rejected |

### 8. State machine — **hand-rolled enum + transition table**

**Chosen:** `emmy_serve/swap/state.py`:

```python
from enum import Enum

class SidecarState(Enum):
    STOPPED = "stopped"
    STARTING = "starting"
    READY = "ready"
    SWAPPING = "swapping"
    DRAINING = "draining"
    ERROR = "error"

# D-07 LOCKED transitions (any state → ERROR is implicit):
ALLOWED_TRANSITIONS: dict[SidecarState, set[SidecarState]] = {
    SidecarState.STOPPED:  {SidecarState.STARTING, SidecarState.ERROR},
    SidecarState.STARTING: {SidecarState.READY, SidecarState.ERROR},
    SidecarState.READY:    {SidecarState.SWAPPING, SidecarState.DRAINING, SidecarState.ERROR},
    SidecarState.SWAPPING: {SidecarState.READY, SidecarState.ERROR},
    SidecarState.DRAINING: {SidecarState.STOPPED, SidecarState.ERROR},
    SidecarState.ERROR:    {SidecarState.STARTING},  # recovery via /start only
}

class StateMachine:
    def __init__(self) -> None:
        self._state = SidecarState.STOPPED
        self._lock = asyncio.Lock()

    async def transition_to(self, new: SidecarState) -> None:
        async with self._lock:
            if new not in ALLOWED_TRANSITIONS[self._state] and new != SidecarState.ERROR:
                raise InvalidTransitionError(self._state, new)
            self._state = new

    @property
    def state(self) -> SidecarState:
        return self._state
```

~30 LOC, fully testable, inspectable from `/status`. Async lock prevents transition races between `/start`, `/stop`, `/profile/swap` handlers running concurrently in uvicorn.

**Alternatives considered:**

| Option | Why rejected |
|--------|--------------|
| `transitions` 0.9.3 | Callback model + decorator declarations are overkill for 6 states. Adds ~500 KB dep. `[CITED: pytransitions/transitions]` is great for complex FSMs (signaling + nested states + queued transitions); we have none of that. |
| `python-statemachine` 3.0.0 | Same overkill argument; nicer DSL but ours fits in a dict. `[CITED: python-statemachine.readthedocs.io]` |
| Plain `str` state | No type safety; typos in transitions go undetected. Enum is one extra line. |

# emmy_serve/swap/orchestrator_runner.py

async def run_orchestrator_subprocess(
    *,
    from_path: str | None,  # None for cold start (no --from arg)
    to_path: str,
    port: int = 8002,
    cwd: str | None = None,
) -> AsyncIterator[dict]:
    """Spawn `uv run python -m emmy_serve.swap.orchestrator ...` and yield parsed JSON-per-line records.

    Final yield is `{"_internal_exit": <int>}` so the caller can route exit codes.
    Caller is responsible for SSE framing.
    """
    argv = ["uv", "run", "python", "-m", "emmy_serve.swap.orchestrator", "--to", to_path, "--port", str(port)]
    if from_path is not None:
        argv.extend(["--from", from_path])
    proc = await asyncio.create_subprocess_exec(
        *argv,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE,
        cwd=cwd,
    )
    assert proc.stdout is not None
    async for line in proc.stdout:
        text = line.decode(errors="replace").strip()
        if not text:
            continue
        try:
            yield json.loads(text)
        except json.JSONDecodeError:
            continue  # silently ignore malformed (matches TS-side discipline)
    rc = await proc.wait()
    yield {"_internal_exit": rc}

### Backoff/retry policy in HTTP-mode (Claude's discretion #4 — resolved)

- **Initial connect:** 5 s connect timeout. On `ECONNREFUSED` (sidecar not running), **fail fast** — return `{exit: 1}` with a clear error message; the user sees "sidecar unreachable; is emmy-sidecar enabled? `systemctl --user status emmy-sidecar`". No retry — sidecar should always be up; bouncing retries would mask real outages.
- **Mid-stream:** No reconnect. SSE drops mid-swap = swap state is unknown → return `{exit: 1}` and let the user re-issue `/status` (which is poll-only and idempotent) to learn current state. This matches the TS-side `runSwapAndStreamProgress` failure behavior (which doesn't retry on subprocess crash either).
- **Read timeout:** None during streaming (the stream may go silent for ≤ 30 s during weight loading; the 15 s sse-starlette ping keeps the socket alive). If 60 s pass with NO ping AND no event, abort.

### Polling cadence (Claude's discretion #5 — resolved)

Remote-mode metrics-poller polls `/status` at the SAME 1 Hz cadence as today's local-mode `nvidia-smi` + vLLM `/metrics` calls (D-23 from Phase 3). Sidecar caches `nvidia-smi` for 1 s internally (per CONTEXT D-07) so the actual subprocess fire-rate is bounded regardless of how many clients poll. Remote-mode could safely go to 2 Hz with no sidecar load increase, but 1 Hz preserves the parity claim and matches user expectations.

   - Tie-breaker for planner: if any operator already runs OTHER services on Tailscale Serve and wants to consolidate URLs, path-prefix wins. Otherwise, separate ports are simpler.

2. **Sidecar `/profiles` endpoint for remote-mode autocomplete? (NOT in scope but worth flagging)**
   - Currently autocomplete in `slash-commands.ts:256` walks the local `profiles/` directory. In remote mode, the Mac has no `profiles/` directory.
   - Workaround for v1: Mac user types variant from memory or reads runbook.
   - Out-of-scope per CONTEXT (deferred to "Cross-machine profile bundles"); planner just needs to confirm the v1 UX is acceptable.

3. **Should the sidecar's `/start`/`/stop` SSE final-record also emit the orchestrator's exit envelope `{rolled_back, rollback_succeeded}` for `/start` (when started cold + boot failed)?**
   - Recommendation: YES — same envelope shape as `/profile/swap`. Lets the TS dispatcher uniformly handle exit-6 across all three lifecycle commands.

4. **systemd unit install: in `start_emmy.sh` (server side, run on Spark) OR in a new `scripts/install_sidecar.sh` script?**
   - `start_emmy.sh` is currently for booting vLLM; adding a one-time install step is a small intent shift.
   - Recommendation: add a `--install-sidecar-unit` flag to `start_emmy.sh` (idempotent — if already installed, no-op). Avoids creating a new script for one operation.

5. **Should the sidecar disable FastAPI's `/docs` and `/redoc` over Tailscale Serve?**
   - Yes — example signature above sets `docs_url=None, redoc_url=None`. CONTEXT defers OpenAPI exposure to a runbook discussion; defaulting to disabled keeps the tailnet attack surface minimal. Operators can flip them on locally for development by editing the FastAPI() construction.
