# Phase 04.2 Plan 05: Mac client install + EMMY_SEARXNG_URL override Summary

**One-liner:** Wires EMMY_SEARXNG_URL env override into web-search.ts (D-33 LOCKED loopback default preserved), expands install-client.sh wrapper with 3 new env vars + Spark-side Tailscale Serve reminders, and adds a D-33 LOCKED air-gap regression canary smoke test.

## What Shipped

After this plan, an operator can run `bash scripts/install-client.sh` on a fresh Mac and get a working `~/.local/bin/emmy` wrapper that routes inference, sidecar control, AND web_search to Spark over Tailscale. The local-mode air-gap thesis is preserved by canary smoke test.

### Task 1: web-search.ts EMMY_SEARXNG_URL override (TDD: RED → GREEN)

**Modified:** `packages/emmy-tools/src/web-search.ts` (+23 LOC)
**Created:** `packages/emmy-tools/tests/web-search-env-override.test.ts` (4 tests, 116 LOC)

The `DEFAULT_CFG.baseUrl` field was changed from a const literal to a **getter** that calls `_resolveDefaultBaseUrl()` returning `process.env.EMMY_SEARXNG_URL ?? "http://127.0.0.1:8888"`. The getter pattern (vs. module-load-time literal evaluation) was chosen per checker iter-1 BLOCKER #5: env vars set after module load (typical in test setup) are invisible to a literal evaluated at import time, forcing fragile cache-busting in tests. The getter resolves env LIVE on every read.

A new `__getDefaultCfgForTests` test export (mirroring the existing `__resetSearchCountForTests` pattern) lets tests assert the resolved baseUrl without exporting `DEFAULT_CFG` itself to production callers.

**Tests (4):**
1. EMMY_SEARXNG_URL set → DEFAULT_CFG.baseUrl uses env override; webSearch fetch hits the env URL (Bun.serve spy on `q=` query param)
2. EMMY_SEARXNG_URL UNSET → DEFAULT_CFG.baseUrl is exactly `http://127.0.0.1:8888` (**D-33 LOCKED canary**)
3. EMMY_SEARXNG_URL set to https Tailscale URL → DEFAULT_CFG.baseUrl matches verbatim
4. env-flip mid-test → getter is live, not cached (BLOCKER #5 determinism guard)

**Untouched:** EMMY_WEB_SEARCH=off kill-switch at line ~244 (orthogonal — disables the tool entirely; this plan only changes the URL resolver).

### Task 3: tests/smoke/verify_airgap_local_mode.sh

**Created:** `tests/smoke/verify_airgap_local_mode.sh` (43 LOC)

The D-33 LOCKED regression canary. Runs `emmy_serve.airgap.ci_verify_phase3 --dry-run` with `EMMY_REMOTE_CLIENT` unset, asserting that the STRICT validator continues to exit 0 after Phase 04.2's web-search.ts override lands. ## Threat Mitigation

- **T-04.2-S4** (EMMY_SEARXNG_URL redirected): Mitigated via trust-on-tailnet (C-05) — install-client.sh composes the URL from operator-supplied `$EMMY_SPARK_HOST` and pins `https://...:8888`. No URL-scheme allowlist added in v1 (over-engineering for single-user single-tailnet); documented in CONTEXT.md.
- **T-04.2-S6** (wrapper env leakage): Mitigated by `exec env … emmy "$@"` form — env vars are inline arguments to `exec`, not exported in the parent shell, so shell history doesn't capture them.
- **T-04.2-AirGap1** (silent override of loopback default): Mitigated by literal default `"http://127.0.0.1:8888"` when env unset; smoke test (Task 3) gates regression.
- **T-04.2-AirGap2** (future PR removing the fallback): Mitigated by test #2 in `web-search-env-override.test.ts` — any code change that breaks the default fails this test loudly.
