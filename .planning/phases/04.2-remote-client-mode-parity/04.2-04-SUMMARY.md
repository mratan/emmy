## Decisions Made

- **Self-contained sidecar-lifecycle-client.ts (NEW)** instead of refactoring Plan 03's `profile-swap-runner-http.ts` — closes BLOCKER #1 from checker iter-1: Plan 03 tests un-invalidated, zero cross-plan files_modified overlap, single-responsibility per HTTP endpoint module. ~30 LOC of duplicated SSE-consume idiom is the cost; cheaper than the test-invalidation + cross-plan merge-conflict cost.
- **register{Start,Stop,Status}Command register UNCONDITIONALLY** (no `opts.profileDir` gate) because the sidecar is the source of truth for valid profiles + lifecycle state — Mac client doesn't have profile bundles by default. The 3 commands work the moment a sidecar URL is reachable.
- **Lazy import** inside both `pi-emmy-extension.ts` factory wiring AND inside metrics-poller's `getStatus` default — keeps local-mode (the daily-driver) at zero import cost; Bun's module cache makes subsequent remote-mode calls O(1).
- **EMMY_REMOTE_CLIENT === "1" literal-string check** (not truthy) — `'0'`, `'true'`, `'false'`, `'yes'` all keep the local path. Matches Plan 03's discipline so the same env value drives both dispatcher branches consistently. Snapshot test in Plan 03 protects this.
- **GPU/tok render `--` in v1 remote mode** — `gpu_temp_c` is temperature not utilization (not directly mappable to existing `gpuPct` field); `/status` v1 has no tok-rate source. Researcher-noted deferral; future plan can widen `FooterValues` with a temperature column or add a sidecar-side rate computation.
- **T-04.2-S2 mitigation in renderSidecarStatus** — `last_error.msg.slice(0, 40)` prevents a malicious / unbounded sidecar error string from line-wrapping the TUI; `Math.round` on numerics is the second guard (sidecar's Pydantic schema already rejects NaN at JSON-parse time).
- **S-2 env-restore via beforeEach/afterEach** in `metrics-poller.remote.test.ts` — handles the `undefined` sentinel correctly (delete vs assign). Required because `EMMY_REMOTE_CLIENT` is process-global and tests must not leak.
