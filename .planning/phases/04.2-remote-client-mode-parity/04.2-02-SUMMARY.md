## Decisions Made

- **TimeoutStopSec=40** — explicit margin over D-01 35s window (30s grace + 5s SIGKILL wait); systemd's default 90s would mask hangs.
- **Linger check is WARNING, not error** — single-user box may have persistent SSH/console session and may not need lingering; RESEARCH risk-table mitigation. Operator sees stderr WARNING + actionable command.
- **Help-selector fix as Rule 1 bug** — pre-existing `sed -n '1,16p'` truncated lines 17-18 of docstring; new Usage line forces re-count to 19, which is also the cleanest moment to fix the pre-existing truncation. This is a Rule 1 (bug) deviation per the GSD deviation rules — the existing behavior was incorrect (`--help` was hiding the Exit-codes block).
- **uv run --extra dev pytest** — system pytest (`/home/mratanap/.local/bin/pytest`) shebangs to `/usr/bin/python3` which has a broken pydantic-core import from a shared NVIDIA venv; `uv run --extra dev pytest` resolves to the project `.venv` with pydantic-core matching pydantic 2.x. Documented in this Summary so future plans don't trip on the same issue.
- **Monkeypatch BOTH the source module AND controller's imported binding** — controller does `from .sidecar_metrics import fetch_vllm_metrics_cached` which creates a separate name binding; cache-refresh code uses the controller-side binding. Patching only sidecar_metrics is insufficient. Pattern documented in test docstring.
- **Per-test isolation via _reset_runtime_for_tests + _reset_caches_for_tests** — both BEFORE and AFTER yield (autouse fixture); safety belt for cross-test isolation. The Plan 01 unit tests use the same pattern.

### Test Infrastructure Discoveries

**2. [Rule 3 - Blocking] System pytest resolves to broken /usr/bin/python3 environment**
- **Found during:** Task 2 (initial `uv run pytest tests/integration/test_sidecar_no_vllm.py` failed at collection with `ImportError: cannot import name 'validate_core_schema' from 'pydantic_core'`)
- **Issue:** `/home/mratanap/.local/bin/pytest` is shebanged `#!/usr/bin/python3` and resolves through `/home/srpost/shared/nvidia-venv/lib/python3.12/dist-packages/` which has fastapi 0.112+ but pydantic 2.6 paired with pydantic-core 2.x where `validate_core_schema` no longer exists.
- **Fix:** Use `uv run --extra dev pytest …` which forces resolution into the project `.venv` (built by `uv` earlier) with the correct pydantic-core. The `--extra dev` flag pulls in the [project.optional-dependencies] dev = [pytest, pytest-asyncio, pyyaml] block from pyproject.toml.

- **Verification:** `uv run --extra dev pytest tests/integration/test_sidecar_no_vllm.py --run-integration -q` → 6 passed in 0.51s.
