## Decisions Made

- **Module-level _DRAIN_GRACE_S / _SIGKILL_WAIT_S overridable via monkeypatch** rather than patching `time.monotonic` globally — patching the time module corrupts asyncio's event-loop time tracking, leading to test hangs.
- **Subprocess IPC to orchestrator** (vs. module import) — preserves Phase-4 main()/sys.exit exit-code path verbatim; consistent with TS-side spawnFn pattern; clean process boundary at the orchestrator-runner seam.
- **_VLLM_BASE_URL configurable via env** (`EMMY_VLLM_URL`) — supports test isolation in environments where another service occupies port 8002 (we observed a stray Prometheus exporter on the worktree host during test execution).
- **`docs_url=None, redoc_url=None`** in FastAPI app construction — security posture per RESEARCH §5; no Swagger/ReDoc surface exposed via Tailscale Serve.
- **`asyncio.create_subprocess_exec`** (NOT `shell=True`) for orchestrator subprocess — eliminates shell-injection threat at the orchestrator boundary (T-04.2-S3).
- **Pydantic v2 `ConfigDict(populate_by_name=True)` + `Field(alias="from")`** for SwapRequest — handles the Python keyword collision cleanly without renaming the wire format.
- **Test pattern: `client.post` rather than `client.stream` + `iter_lines`** for SSE responses — TestClient buffered post returns the entire response body once the server closes the connection, avoiding hangs caused by sse-starlette's ping keepalive.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] orchestrator.py main() required `--from` but cold-start argv omits it**
- **Found during:** Task 3 (cold-start path implementation)
- **Issue:** The plan specifies cold-start argv as `["uv", "run", "python", "-m", "emmy_serve.swap.orchestrator", "--to", to_path, "--port", str(port)]` — no `--from` arg. But `emmy_serve/swap/orchestrator.py:main()` had `--from` as `required=True`, so the cold-start subprocess would fail argparse before swap_profile() runs.
- **Fix:** Made `--from` optional (`required=False, default=None`). When omitted, main() now passes `Path()` (empty) as `old_profile` and forces `no_rollback=True` (cold start has no engine to roll back to). swap_profile()'s docker stop step uses `check=False`, so an empty `old_profile` is harmless. Phase-4 contract for the WITH-from path is unchanged.
- **Files modified:** emmy_serve/swap/orchestrator.py (argparse `--from` clause + main() body)
- **Verification:** Existing tests/unit/test_swap_preflight_fail.py (call swap_profile directly, not via main()) continue to pass; full unit suite green
- **Committed in:** cf13e99 (Task 3 commit)

**2. [Rule 3 - Blocking] Test isolation required _VLLM_BASE_URL override pattern**
- **Found during:** Task 3 (initial /status test had vllm_up=True even with no vLLM running)
- **Issue:** The test environment had an unrelated service on 127.0.0.1:8002 returning a Prometheus /metrics endpoint, causing `test_status_payload_schema_when_stopped` to assert vllm_up=False but receive vllm_up=True. This is environment leakage, not a controller bug.
- **Fix:** Updated `_client_with_no_vllm` test helper in test_sidecar_status.py to monkeypatch `fetch_vllm_metrics_cached` to raise ConnectionRefusedError. This deterministically simulates "vLLM not running" regardless of host environment.
- **Files modified:** tests/unit/test_sidecar_status.py
- **Verification:** All 4 /status endpoint tests now green; no host-environment dependence
- **Committed in:** cf13e99 (Task 3 commit)

**3. [Rule 3 - Blocking] Test SSE consumption pattern (`client.stream` + `iter_lines`) hangs on sse-starlette keepalive**
- **Found during:** Task 3 (test_sidecar_stop.py initial implementation timed out at 30s)
- **Issue:** TestClient.stream + iter_lines blocks waiting for the SSE response body to close. sse-starlette's `ping=15` keepalive keeps the connection open longer than the test wants. The /stop drain loop also wanted patched time.monotonic, but module-level patching of `_ctl.time.monotonic` corrupted asyncio's event-loop time, causing the loop to hang on `await asyncio.sleep`.
- **Fix:** Switched to `client.post` (TestClient buffers the full response body once the server closes the connection — which our generator does after yielding the final frame); added `_data_frames_from_text` parser. Also moved drain/sigkill timing thresholds to module-level constants (`_DRAIN_GRACE_S`, `_SIGKILL_WAIT_S`) overridable via monkeypatch — preferred over patching time.monotonic globally.
- **Files modified:** tests/unit/test_sidecar_stop.py (consume helper); emmy_serve/swap/controller.py (added module-level _DRAIN_GRACE_S, _SIGKILL_WAIT_S constants — already present in initial implementation; just relied on by tests)
- **Verification:** All 6 stop tests green in <1s; no flake
- **Committed in:** cf13e99 (Task 3 commit)
