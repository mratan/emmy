# Phase 04.5 Plan 03: OTel parent-child propagation Summary

`withAgentToolSpan` + `withSubagentSpan` helpers establish the LOCKED 4-level trace tree
(parent_session → agent.tool.Agent → subagent.\<name> → child HTTP). The Agent tool's
execute owns the Level 2 tool-call span (W1 fix — required for the 4-level shape);
the dispatcher owns Level 3. Failure paths fire `recordException` + `setStatus(ERROR)`
on both span sites so sub-agent failures are NEVER black-boxes (Pitfall #18 mitigation).

**Files:** 2 created (otel.ts + test), 4 modified (dispatcher, index, types, package.json)
**Tests:** 8 new (5 helper + 1 W1 W1 test 6 + 1 W1 chain test 7 + 1 V4 E2E) — all green
**Total subagent test count:** 22 pass / 0 fail / 68 expect calls
**Typecheck:** clean

## What was built

### Source

- `otel.ts`:
  - `SUBAGENT_TRACER_NAME = "emmy-subagent"` — tracer identity for both helpers.
  - `withAgentToolSpan(personaName, parentSessionId?, inner)` opens `agent.tool.Agent`
    span with attrs: `gen_ai.tool.name="Agent"`, `gen_ai.tool.persona=<personaName>`,
    `gen_ai.conversation.id=<sid>`. (W1 — Level 2 of LOCKED tree.)
  - `withSubagentSpan({name, pattern}, parentSessionId?, inner)` opens
    `subagent.<persona.name>` span with attrs: `gen_ai.agent.name`, `gen_ai.agent.id`
    (v1 fallback = name), `gen_ai.conversation.id`, `emmy.subagent.pattern`,
    `emmy.subagent.parent_span_id` (informational). (Level 3.)
  - Both helpers' inner is wrapped in try/catch/finally: on throw, `recordException`
    + `setStatus(SpanStatusCode.ERROR)` + propagate; finally always `span.end()`.

- `dispatcher.ts`:
  - Public `dispatchSubAgent` now wraps the body in `withSubagentSpan(...)` and
    sets `emmy.subagent.final_text_chars = result.output.length` on success.
  - The existing services + spawn + dispose logic moved into a private
    `dispatchSubAgentInner` (no behavioral change — just refactor for span wrapping).

- `index.ts`:
  - Tool's `execute` wraps `dispatchSubAgent(...)` call in `withAgentToolSpan(persona.name, opts.parentSessionId, ...)` (W1).

- `types.ts`:
  - `CreateSubAgentToolOpts` gains optional `parentSessionId?: string` (the parent's
    pi AgentSession id; stamped on `gen_ai.conversation.id` for both span sites).

### Notes on OTel 2.x compatibility

- `BasicTracerProvider.register()` was REMOVED in OTel SDK 2.x. Tests must call
  `trace.setGlobalTracerProvider(provider)` AND `context.setGlobalContextManager(new AsyncHooksContextManager().enable())` explicitly. Without the context manager, `startActiveSpan` does NOT activate the new span across `await` boundaries, and `traceId` propagation breaks (the failure mode we caught and fixed mid-test-write).

## Deviations from Plan

**1. [Rule 1 — bug] BasicTracerProvider.register() doesn't exist in OTel SDK 2.x.**
   - Found during: Test 1 first run.
   - Fix: switched to `trace.setGlobalTracerProvider(provider)` per OTel 2.x migration guide.

**2. [Rule 1 — bug] Spans on different traces — async context not threaded through awaits.**
   - Found during: Test 7 first run after fix #1.
   - Fix: added `@opentelemetry/context-async-hooks` 2.1.0 as devDep + `context.setGlobalContextManager(new AsyncHooksContextManager().enable())` in `beforeAll`. AsyncLocalStorage now carries the active span across awaits.

**3. [Rule 3 — additional functionality] Refactored dispatcher into outer + inner.**
   - Found during: wiring `withSubagentSpan` around the existing body.
   - Fix: split `dispatchSubAgent` into a thin outer that opens the span +
     stamps `final_text_chars`, and `dispatchSubAgentInner` carrying the existing
     services + spawn + dispose logic. No behavioral change vs. Plan 04.5-01.
     Maintains the test seam (`__setSessionFactoryForTests`) — V1 dispose tests
     still all pass.

**Total deviations:** 3 (2 Rule 1 OTel API changes, 1 Rule 3 refactor for clarity).
