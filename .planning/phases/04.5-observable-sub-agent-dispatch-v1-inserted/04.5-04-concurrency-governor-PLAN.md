<objective>
Add the concurrency governor that gates Agent dispatch by `max_concurrent` (default 2 — vLLM-imposed hardware ceiling per the concurrency spike) AND serializes dispatch when parent input tokens exceed `long_context_serialize_threshold_tokens` (default 40000) AND rejects over-cap dispatches with the `agent.dispatch.rejected` telemetry event (I3 fix — pins V5 over-cap behavior). Emit `agent.dispatch.queued`, `agent.dispatch.serialized`, and `agent.dispatch.rejected` telemetry events. Cover V5 (concurrent children, no cross-talk; over-cap rejection) and V6 (long-context serialization) AND **V7 BEHAVIORAL** (B1 fix — auto-compaction-off behavior, NOT just static presence).

Purpose: Without a governor, a parent could fire N parallel Agent dispatches under high load and OOM the vLLM engine (KV cache contention is the hardware ceiling that drove the LOCKED `max_concurrent: 2`). The long-context rule is preventative: a 40K-token parent already consumes >35% of `max_input_tokens` (114688), so parallelizing two more children at high context risks preemption. Pi-mono and the underlying vLLM engine don't enforce this — Emmy must.

Auto-compaction-off (V7) was previously only verified by static `grep` for `setAutoCompactionEnabled(false)` in the dispatcher source (Plan 04.5-01 acceptance). Static presence ≠ behavioral verification — pi could process the call but a future SDK change could ignore it silently. Task 3 of this plan (B1 fix) drives a faux child past pi's `reserveTokens=16384` threshold and asserts no `compaction` event fires — the actual contract Pitfall #18 cares about (sub-agents are short-lived; mid-run compaction would surface unexpected truncation in the parent's tool result).

Output: New `governor.ts` module with semaphore-style acquire/release; dispatcher wraps spawn in acquire; CreateSubAgentToolOpts gains `parentInputTokens` + `governor` fields. New `auto-compaction.test.ts` covering V7 behavioral.

         ```typescript
         import { emitEvent } from "@emmy/telemetry";

         export interface ConcurrencyGovernor {
           acquire(opts: { parentInputTokens?: number }): Promise<{ release: () => void; serialized: boolean; waited_ms: number }>;
         }

         export function createConcurrencyGovernor(config: {
           maxConcurrent: number;
           longContextSerializeThresholdTokens: number;
           rejectOverCap?: boolean;                          // I3 — default true
         }): ConcurrencyGovernor {
           const rejectOverCap = config.rejectOverCap ?? true;     // I3 LOCKED default
           let inFlight = 0;
           const queue: Array<() => void> = [];

           function effectiveCap(parentInputTokens: number | undefined): { cap: number; serialized: boolean } {
             if (parentInputTokens != null && parentInputTokens > config.longContextSerializeThresholdTokens) {
               return { cap: 1, serialized: true };
             }
             return { cap: config.maxConcurrent, serialized: false };
           }

           async function acquire(opts: { parentInputTokens?: number }) {
             const startedAt = performance.now();
             const { cap, serialized } = effectiveCap(opts.parentInputTokens);

             if (serialized) {
               emitEvent({
                 event: "agent.dispatch.serialized",
                 ts: new Date().toISOString(),
                 parent_input_tokens: opts.parentInputTokens!,
                 threshold: config.longContextSerializeThresholdTokens,
               });
             }

             // I3 — over-cap rejection (skip the wait queue entirely if at cap and rejectOverCap is set)
             if (rejectOverCap && inFlight >= cap) {
               emitEvent({
                 event: "agent.dispatch.rejected",
                 ts: new Date().toISOString(),
                 reason: "over_cap",
                 in_flight: inFlight,
                 cap,
               });
               throw new Error(`[Agent] dispatch rejected: concurrent dispatch cap reached (in_flight=${inFlight}, cap=${cap})`);
             }

             // Default queue mode (rejectOverCap=false): wait until capacity is available.
             while (inFlight >= cap) {
               await new Promise<void>((r) => queue.push(r));
             }
             inFlight++;
             const waited_ms = performance.now() - startedAt;
             if (waited_ms > 1) {
               emitEvent({
                 event: "agent.dispatch.queued",
                 ts: new Date().toISOString(),
                 wait_ms: waited_ms,
                 in_flight: inFlight,
                 cap,
               });
             }
             let released = false;
             const release = () => {
               if (released) return;
               released = true;
               inFlight--;
               const next = queue.shift();
               if (next) next();
             };
             return { release, serialized, waited_ms };
           }

           return { acquire };
         }
         ```
