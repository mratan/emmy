# Phase 04.5 Plan 06: Per-child JSONL routing + sidecar forensics index Summary

`resolveChildSessionManager` routes child sessions through `SessionManager.inMemory`
or `SessionManager.create` based on persona's `persistTranscript` flag. Every
dispatch (success or failure) writes a sidecar JSONL entry linking child →
parent OTel span via `parent_span_id` + `trace_id` — Emmy's belt-and-suspenders
to OTel for offline replay/audit (Pitfall #18 mitigation).

**Tests:** 12 new (6 jsonl + 6 sidecar) — all green
**Total subagent tests:** 46 pass / 0 fail / 139 expect calls
**Typecheck:** clean

## What was built

### Source — jsonl.ts

- I2 fix: top-of-file comment block documenting the LOCKED kebab-case (profile
  bundle dir) vs snake_case (yaml key + on-disk parent child-instance dir)
  persona naming convention. Grep-discoverable via "Persona naming convention".
- `resolveChildSessionManager(persona, parentCwd, childSessionId)`:
  - `persistTranscript: false` (or undefined) → `SessionManager.inMemory(parentCwd)` (no disk write).
  - `persistTranscript: true` → `SessionManager.create(parentCwd, subDir)` where `subDir = <parentCwd>/.emmy/subagents/<persona.name>-<iso-ts>-<id8>`.
  - Validates `persona.name` against `/^[a-zA-Z0-9_-]+$/` before using it in path; throws on mismatch.
- `writeSidecarEntry(parentSessionDir, parentSessionId, entry)`:
  - Appends to `<parentSessionDir>/session-<parentSessionId>.subagents.jsonl`.
  - NO-OP if either path component is undefined.
  - Best-effort: filesystem errors logged to stderr but do NOT propagate (sidecar is forensics, not authoritative — OTel span is authoritative).

### Source — dispatcher.ts wiring

- Outer `dispatchSubAgent` restructured to capture sidecar fields across span boundary:
  - `started_at` captured before governor.acquire.
  - `parent_span_id` + `trace_id` captured inside `withSubagentSpan` from `span.spanContext()`.
  - `child_session_id` + `child_jsonl_path` captured via callback from `dispatchSubAgentInner`.
  - On SUCCESS: writeSidecarEntry with `ok: true`, `ended_at: now`.
  - On FAILURE (any throw after governor acquire): writeSidecarEntry with `ok: false`, `ended_at: now`. Skipped if acquire itself rejected (over-cap path) since no child was instantiated.
  - `release()` in `finally` regardless.
- `dispatchSubAgentInner` now uses `resolveChildSessionManager` (replaces bare `SessionManager.inMemory`); pre-generates `childSessionIdSeed = randomUUID()` for the dirname suffix; reads SDK-assigned `child.sessionId` for the sidecar entry's `child_session_id` field.

### Source — types.ts

- `CreateSubAgentToolOpts.parentSessionDir?: string` — base dir for the sidecar index file (typically pi's session dir).
